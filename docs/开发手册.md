```
└────────────┘    └────────────┘    └────────────┘
```

## 二、变量占位符基础

### 2.1 简单变量

Sheet名+字段编码，自动组合为完整路径：

```
Sheet: 全局信息
{{ 全局信息.公司名 }}
{{ 全局信息.成立日期 }}

Sheet: 基本情况
{{ 基本情况.注册资本 | num }}
{{ 基本情况.有职工信息 }}
```

### 2.2 过滤器

对变量值进行格式化：

```
{{ 金额 | money }}        千分位金额（1000 → 1,000.00）
{{ 比例 | percent }}      百分比（小数自动乘100，0.25 → 25.00%）
{{ 比例 | num }}          千分位数字（1000 → 1,000，1000.5 → 1,000.50）
{{ 日期 | date }}          日期格式化（2025-01-15 → 2025年01月15日）
{{ 数量 | int }}          整数
{{ 备注 | default_dash }}  空值显示为空字符串
{{ 备注 | default('-') }} 空值默认
```

## 三、条件显示

### 3.1 段落级条件

整段内容根据条件显示/隐藏：

```
{%p if 显示备注 %}
备注：{{ 备注内容 }}
{%p endif %}
```

**说明**：
- `显示备注` 为 `TRUE`（或 `True`）时显示段落
- 为 `FALSE`（或 `False`）时完全隐藏段落
- Excel 中填写的 `"TRUE"`/`"FALSE"` 字符串会自动转为 Python 布尔值，无需手动转换

### 3.2 行内条件

段落内的条件判断：

```
{% if 折扣 > 0 %}
折扣：{{ 折扣 | money }}元
{% endif %}
```

### 3.3 比较运算

| 运算符 | 说明 | 示例 |
|--------|------|------|
| `==` | 等于 | `{% if g.type == "A" %}` |
| `!=` | 不等于 | `{% if g.status != "draft" %}` |
| `>` | 大于 | `{% if amount > 1000 %}` |
| `<` | 小于 | `{% if count < 10 %}` |
| `and` | 且 | `{% if a and b %}` |
| `or` | 或 | `{% if a or b %}` |

## 四、表格循环

### 4.1 基础循环

用 `{%tr for %}` 循环生成表格行：

```
| 名称 | 数量 | 单价 | 金额 |
{%tr for i in 商品列表 %}
| {{ i.名称 }} | {{ i.数量 }} | {{ i.单价 | money }} | {{ i.金额 | money }} |
{%tr endfor %}
```

**说明**：
- `{%tr for %}` 行会**被移除**
- 中间行根据数据**循环复制**
- `{%tr endfor %}` 行会**被移除**
- Sheet命名为 `商品列表`
- 循环变量名统一使用 `i`

### 4.2 动态行数示例

**模板表格**：

| 序号 | 商品名称 | 规格 | 数量 | 单价 | 金额 |
|------|---------|------|------|------|------|
| {%tr for i in 产品列表 %} | | | | | |
| {{ i.序号 }} | {{ i.名称 }} | {{ i.规格 }} | {{ i.数量 }} | {{ i.单价 | money }} | {{ i.金额 | money }} |
{%tr endfor %} | | | | | |
| | | | | **合计** | **{{ 合计金额 | money }}** |

**Excel数据** (`产品列表` Sheet)：

| 序号 | 名称 | 规格 | 数量 | 单价 | 金额 |
|------|------|------|------|------|------|
| 1 | 产品A | 规格1 | 10 | 100 | 1000 |
| 2 | 产品B | 规格2 | 5 | 200 | 1000 |

**渲染结果**（2行数据）：

| 序号 | 商品名称 | 规格 | 数量 | 单价 | 金额 |
|------|---------|------|------|------|------|
| 1 | 产品A | 规格1 | 10 | 100.00 | 1,000.00 |
| 2 | 产品B | 规格2 | 5 | 200.00 | 1,000.00 |
| | | | | **合计** | **2,000.00** |

### 4.3 空数据处理

数据为空时隐藏整块内容：

```
{%p if 商品列表 %}
商品明细：
| 名称 | 数量 |
{%tr for i in 商品列表 %}
| {{ i.名称 }} | {{ i.数量 }} |
{%tr endfor %}
{%p endif %}
```

## 五、模板结构设计

### 5.1 文档结构建议

```
┌─────────────────────────────────────────┐
│  封面/标题区（如 基本信息 Sheet）        │
├─────────────────────────────────────────┤
│  基本信息区（如 基本情况 Sheet）          │
├─────────────────────────────────────────┤
│  主内容区A（如 商品列表 循环，可能有条件）│
├─────────────────────────────────────────┤
│  主内容区B（如 股东出资 循环，可能有条件）│
├─────────────────────────────────────────┤
│  汇总区（汇总变量）                       │
├─────────────────────────────────────────┤
│  签章区（签章相关变量）                   │
└─────────────────────────────────────────┘
```

### 5.2 命名规范

| 类型 | Sheet命名 | 模板引用 | 用途 |
|------|---------|---------|------|
| 键值对 | `全局信息` | `{{ 全局信息.公司名 }}` | 单个变量值 |
| 表格数据 | `股东出资` | `{%tr for i in 股东出资 %}` | 循环数据列表 |

**Sheet名作为Context Key**：
- Sheet名 = Context key，与模板变量中的第一级路径完全一致
- 模板变量 = `{{ Sheet名.字段编码 }}`
- 循环引用 = `{%tr for i in Sheet名 %}`，循环内字段 = `{{ i.列标题 }}`

### 5.3 命名原则

- Sheet名称使用中文语义名，直接表达数据含义
- 字段编码在Excel中只写字段名，与Sheet名直接对应
- 循环变量统一使用 `i`
- 变量值支持中文内容

## 十、核心API

系统支持 Excel / YAML 两种数据源，统一经 `DataMapper` 处理后由 `DocumentGenerator` 渲染。

### API 速查

| API | 导入路径 | 用途 |
|-----|---------|------|
| `ExcelDataReader` | `src.reader.xlsx` | 读取 .xlsx |
| `YamlDataReader` | `src.reader.yaml` | 读取 .yaml |
| `DataMapper` | `src.processing.mapper` | Excel/YAML → context |
| `DataSchema` | `src.processing.schema` | 数据校验 schema |
| `SchemaValidator` | `src.processing.schema` | 执行校验 |
| `DocumentGenerator` | `src.render.generator` | 模板渲染 |

### 典型用法

```python
from src.reader.xlsx import ExcelDataReader
from src.reader.yaml import YamlDataReader
from src.processing.mapper import DataMapper
from src.render.generator import DocumentGenerator

# Excel 路径
raw = ExcelDataReader("data.xlsx").read_all()
ctx = DataMapper(raw).build_context()

# YAML 路径（同样走 Mapper 做值转换）
raw = YamlDataReader("data.yaml").read_all()
ctx = DataMapper(raw).build_context()

# 渲染
gen = DocumentGenerator("template.docx")
gen.render(ctx, "output.docx")

# 检查未定义变量
undeclared = gen.get_undeclared_variables(ctx)
```

### 嵌套数据结构（v2.0）

DataMapper 支持通过 Sheet 命名表达三层嵌套。含点号的 Sheet 名会被解析为嵌套关系。

```python
# Sheet "项目公司" → 父列表，Sheet "项目公司.股东出资" → 子列表
raw = {
    "项目公司": pd.DataFrame({"公司简称": ["重庆晟和泰", "安徽富军"]}),
    "项目公司.股东出资": pd.DataFrame({
        "_parent_公司简称": ["重庆晟和泰", "重庆晟和泰", "安徽富军"],
        "股东": ["张伟", "李芳", "王强"],
    }),
}
ctx = DataMapper(raw).build_context()
# ctx["项目公司"][0]["股东出资"] → [{"股东": "张伟"}, {"股东": "李芳"}]
```

**子表关联规则**：
- `父.子`：子表首列必须 `_parent_` 开头，值匹配父表同名列
- 键值对子表自动挂载为 dict，表格子表自动挂载为 list
- 多层嵌套用 `_parent_` 逐层匹配；匹配失败抛 `DataReadError`

### YAML 嵌套

```yaml
项目公司:
  - 公司简称: 重庆晟和泰
    股东出资:
      - 股东: 张伟
        认缴额: 600
```
YAML 天然支持嵌套，同样经过 `DataMapper.build_context()` 做值转换（None→''、TRUE/FALSE→bool）。

## 十一、自定义过滤器

过滤器定义在 `src/render/filters.py`，直接向 `FILTERS` 字典添加即可。

```python
# src/render/filters.py

def filter_rmb(value: Any) -> str:
    if value is None or value == '':
        return ''
    try:
        return f"¥{float(value):,.2f}"
    except (ValueError, TypeError):
        return str(value)

FILTERS['rmb'] = filter_rmb
```

模板中使用：

```
金额：{{ amount | rmb }}
```

## 十二、扩展开发

### 12.1 添加新数据类型

如需支持更多数据格式（如JSON、CSV），创建新的Reader：

```python
class JsonDataReader:
    """JSON数据读取器"""
    
    def __init__(self, file_path: str):
        self.file_path = file_path
    
    def read_all(self) -> Dict[str, Any]:
        import json
        with open(self.file_path, 'r') as f:
            return json.load(f)
```

### 12.2 添加数据验证

扩展 `DataMapper` 添加验证逻辑：

```python
class ValidatedDataMapper(DataMapper):
    """带验证的数据映射器"""

    REQUIRED_FIELDS = ['全局信息.公司名', '基本情况.注册资本']

    def validate(self) -> List[str]:
        """验证数据完整性"""
        errors = []
        for field in self.REQUIRED_FIELDS:
            if not self._has_field(field):
                errors.append(f"缺少必填字段: {field}")
        return errors
```

### 12.3 自定义渲染器

扩展 `DocumentGenerator` 添加后处理：

```python
class AdvancedDocumentGenerator(DocumentGenerator):
    """高级文档生成器"""

    def render_with_postprocess(self, context, output_path):
        self.render(context, output_path)
        self._update_toc(output_path)
        self._add_watermark(output_path)
```

# 第三篇：模板替换规范

## 十九、概述

本文档定义了通用的Word文档（.docx）转换为Jinja2模板（docxtpl格式）的替换规范。适用于任何需要将固定文本转换为可编程变量的场景，如：

- 审计报告自动化生成
- 合同批量生成
- 证书/证明批量制作
- 各类标准化文档生成

---

## 二十、核心概念

### 20.1 模板引擎选择

| 引擎 | 适用场景 | 特点 |
|------|---------|------|
| **docxtpl** | Word文档模板 | 基于Jinja2，保留Word格式，支持循环/条件 |
| **python-docx** | 程序化创建Word | 从零构建文档，精细控制格式 |

**推荐**：使用 **docxtpl** 进行模板驱动生成，业务人员可直接在Word中编辑模板。

### 20.2 Jinja2语法简介

```jinja2
{{ variable }}           # 变量输出
{{ variable | filter }}  # 带过滤器的变量
{% if condition %}       # 条件判断
{% for item in list %}   # 循环
{% endif %}
{% endfor %}
```

**docxtpl扩展语法**：

| 语法 | 用途 | 示例 |
|------|------|------|
| `{%p if %}` | 段落级条件 | `{%p if show_section %}内容{%p endif %}` |
| `{%tr for %}` | 表格行循环 | `{%tr for item in items %}{{ item.name }}{%tr endfor %}` |
| `{%tc for %}` | 表格列循环 | 较少使用 |
| `{{r var }}` | 富文本变量 | 支持动态样式 |

---

## 二十一、变量命名规范

### 21.1 命名原则

采用 **层级命名法**，Sheet名即Context key：

```
Sheet: 全局信息
模板: {{ 全局信息.公司名 }}
模板: {{ 全局信息.注册资本 | num }}

Sheet: 股东出资
模板: {%tr for i in 股东出资 %}
       {{ i.认缴金额 | money }} | {{ i.认缴比例 | percent }}
       {%tr endfor %}
```

**命名规则**：
1. Sheet名直接作为模板变量的第一级路径（Context key）
2. 字段编码在Excel中只写字段名，系统自动按Sheet名组合为完整路径
3. 变量名支持中文（如 `公司名`、`注册资本`、`认缴金额`）
4. 保持命名一致性

### 21.2 数据模型说明

系统采用**扁平数据模型**，Sheet名直接映射为Context的一级key：

| 数据结构 | Sheet名 | 模板引用方式 |
|---------|---------|-------------|
| 键值对 | 语义名（如 `全局信息`） | `{{ 全局信息.字段名 }}` |
| 表格列表 | 语义名（如 `股东出资`） | `{%tr for i in 股东出资 %}` |

不再需要在Sheet名中添加前缀区分数据类型，系统根据Sheet内容自动推断（DataFrame→列表，dict→键值对）。

### 21.3 命名示例

**推荐的中文命名**：
```jinja2
{{ 全局信息.公司名 }}
{{ 基本情况.注册资本 | num }}
{{ 基本情况.有职工信息 }}        # 布尔标志，控制条件显示
{%tr for i in 股东出资 %}
| {{ i.股东 }} | {{ i.认缴金额 | money }} |
{%tr endfor %}
```

**说明**：
- `全局信息`、`基本情况` 等为Sheet名，由用户自定义中文名称
- 字段编码（如 `公司名`、`注册资本`）在Excel中只写字段名即可
- 布尔标志（如 `有职工信息`）填写 `TRUE`/`FALSE` 控制条件渲染

### 22.3 布尔值处理

Excel 中填写的布尔值会自动转为 Python 布尔值：

| Excel填写 | Python值 | 条件渲染结果 |
|-----------|---------|------------|
| `TRUE` | `True` | 显示 |
| `FALSE` | `False` | 隐藏 |
| `true` / `TRUE` | `True` | 显示 |
| 空字符串 `''` | `''` | 隐藏（空值） |

**注意**：`"FALSE"` 作为字符串时在 Jinja2 中为真值，会始终显示。系统会自动将 Excel 中的 `"TRUE"` / `"FALSE"` 字符串转换为 Python `True` / `False`，无需手动处理。

## 二十四、自定义过滤器

### 24.1 常用过滤器

| 过滤器 | 功能 | 示例 |
|--------|------|------|
| `money` | 千分位金额 | `{{ 1234567.89 \| money }}` → "1,234,567.89" |
| `percent` | 百分比 | `{{ 0.1567 \| percent }}` → "15.67%"（自动乘100） |
| `num` | 千分位数字 | `{{ 1234567 \| num }}` → "1,234,567" |
| `date` | 日期格式化 | `{{ g.date \| date }}` → "2025年01月15日" |
| `default_dash` | 空值转字符串、`None`→`""` | `{{ data.x \| default_dash }}` |

### 24.2 过滤器实现

```python
def format_money(value):
    """金额格式化"""
    if value is None or value == '':
        return ''
    try:
        num = float(value)
        if num == 0:
            return ''
        return f"{num:,.2f}"
    except (ValueError, TypeError):
        return str(value)

def format_percent(value):
    """百分比格式化"""
    if value is None or value == '':
        return ''
    try:
        return f"{float(value) * 100:.2f}%"  # 小数自动乘100
    except (ValueError, TypeError):
        return str(value)

def format_num(value):
    """千分位数字格式化"""
    if value is None or value == '':
        return ''
    try:
        num = float(value)
        if num == 0:
            return ''
        if num == int(num):
            return f"{int(num):,}"   # 整数无小数
        return f"{num:,.2f}"         # 小数保留两位
    except (ValueError, TypeError):
        return str(value)
```

## 二十七、最佳实践

### 27.1 模板设计原则

1. **分离内容与格式**：样式定义在Word模板中，数据通过变量注入
2. **变量命名清晰**：使用有意义的变量名，避免缩写
3. **条件控制粒度**：段落级用 `{%p if %}`，行内用标准 `{% if %}`
4. **数据验证**：生成前校验必填字段和数据类型

### 27.2 常见问题

**Q1: 替换后格式丢失？**  
A: 确保使用Run合并策略，不要直接操作单个Run。

**Q2: Jinja2标签显示为纯文本？**  
A: 检查标签是否被Word拆分到多个Run中，使用合并策略解决。

**Q3: 动态表格格式错乱？**  
A: `{%tr for %}` 和 `{%tr endfor %}` 各占独立的表格行，循环体在二者之间。

**Q4: 条件判断不生效？**
A: 检查布尔值是否填写了 `TRUE`/`FALSE`（不区分大小写），系统会自动转换为 Python 布尔值。
