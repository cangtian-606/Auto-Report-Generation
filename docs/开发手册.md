## 二、变量占位符基础

### 2.1 简单变量

在Word中直接输入变量占位符：

```
{{ date.公司名称 }}
{{ date.日期 }}
```

### 2.2 过滤器

对变量值进行格式化：

```
{{ date.金额 | money }}        金额千分位
{{ date.比例 | percent }}      百分比
{{ date.数量 | int }}          整数
{{ date.备注 | default('-') }}  空值默认
```

## 三、条件显示

### 3.1 段落级条件

整段内容根据条件显示/隐藏：

```
{%p if date.显示备注 %}
备注：{{ date.备注内容 }}
{%p endif %}
```

**说明**：
- `date.显示备注` 为 `TRUE` 时显示段落
- 为 `FALSE` 时完全隐藏段落

### 3.2 行内条件

段落内的条件判断：

```
{% if form.折扣 > 0 %}
折扣：{{ form.折扣 | money }}元
{% endif %}
```

### 3.3 比较运算

| 运算符 | 说明 | 示例 |
|--------|------|------|
| `==` | 等于 | `{% if g.type == "A" %}` |
| `!=` | 不等于 | `{% if g.status != "draft" %}` |
| `>` | 大于 | `{% if data.amount > 1000 %}` |
| `<` | 小于 | `{% if data.count < 10 %}` |
| `and` | 且 | `{% if a and b %}` |
| `or` | 或 | `{% if a or b %}` |

## 四、表格循环

### 4.1 基础循环

用 `{%tr for %}` 循环生成表格行：

```
| 名称 | 数量 | 单价 | 金额 |
{%tr for item in form.商品列表 %}
| {{ item.名称 }} | {{ item.数量 }} | {{ item.单价 | money }} | {{ item.金额 | money }} |
{%tr endfor %}
```

**说明**：
- `{%tr for %}` 行会**被移除**
- 中间行根据数据**循环复制**
- `{%tr endfor %}` 行会**被移除**
- Sheet命名为 `form.商品列表`

### 4.2 动态行数示例

**模板表格**：

| 序号 | 商品名称 | 规格 | 数量 | 单价 | 金额 |
|------|---------|------|------|------|------|
| {%tr for row in form.产品列表 %} | | | | | |
| {{ row.序号 }} | {{ row.名称 }} | {{ row.规格 }} | {{ row.数量 }} | {{ row.单价 | money }} | {{ row.金额 | money }} |
{%tr endfor %} | | | | | |
| | | | | **合计** | **{{ form.合计金额 | money }}** |

**Excel数据** (`form.产品列表` Sheet)：

| 序号 | 名称 | 规格 | 数量 | 单价 | 金额 |
|------|------|------|------|------|------|
| 1 | 产品A | 规格1 | 10 | 100 | 1000 |
| 2 | 产品B | 规格2 | 5 | 200 | 1000 |

**渲染结果**（2行数据）：

| 序号 | 商品名称 | 规格 | 数量 | 单价 | 金额 |
|------|---------|------|------|------|------|
| 1 | 产品A | 规格1 | 10 | 100.00 | 1,000.00 |
| 2 | 产品B | 规格2 | 5 | 200.00 | 1,000.00 |
| | | | | **合计** | **2,000.00** |

### 4.3 空数据处理

数据为空时隐藏整块内容：

```
{%p if form.商品列表 %}
商品明细：
| 名称 | 数量 |
{%tr for item in form.商品列表 %}
| {{ item.名称 }} | {{ item.数量 }} |
{%tr endfor %}
{%p endif %}
```

## 十一、自定义过滤器

### 11.1 注册过滤器

在 `TemplateRenderer` 中修改 `_register_filters` 方法：

```python
def _register_filters(self) -> Dict[str, callable]:
    filters = super()._register_filters()
    
    # 添加自定义过滤器
    filters['my_filter'] = self._filter_my_filter
    filters['currency'] = self._filter_currency
    
    return filters

@staticmethod
def _filter_my_filter(value: Any) -> str:
    """自定义过滤器实现"""
    if value is None:
        return ''
    # 处理逻辑
    return str(value)

@staticmethod
def _filter_currency(value: Any, symbol: str = '¥') -> str:
    """货币格式化"""
    if value is None or value == '':
        return f'{symbol}0.00'
    try:
        return f'{symbol}{float(value):,.2f}'
    except:
        return str(value)
```

### 11.2 模板中使用

```
金额：{{ data.amount | currency }}
金额：{{ data.amount | currency('€') }}
```

## 十二、扩展开发

### 12.1 添加新数据类型

如需支持更多数据格式（如JSON、CSV），创建新的Reader：

```python
class JsonDataReader:
    """JSON数据读取器"""
    
    def __init__(self, file_path: str):
        self.file_path = file_path
    
    def read_all(self) -> Dict[str, Any]:
        import json
        with open(self.file_path, 'r') as f:
            return json.load(f)
```

### 12.2 添加数据验证

扩展 `DataMapper` 添加验证逻辑：

```python
class ValidatedDataMapper(DataMapper):
    """带验证的数据映射器"""
    
    REQUIRED_FIELDS = ['g.company_name', 'g.date']
    
    def validate(self) -> List[str]:
        """验证数据完整性"""
        errors = []
        for field in self.REQUIRED_FIELDS:
            if not self._has_field(field):
                errors.append(f"缺少必填字段: {field}")
        return errors
```

# 第三篇：模板替换规范

## 十九、概述

本文档定义了通用的Word文档（.docx）转换为Jinja2模板（docxtpl格式）的替换规范。适用于任何需要将固定文本转换为可编程变量的场景，如：

- 审计报告自动化生成
- 合同批量生成
- 证书/证明批量制作
- 各类标准化文档生成

---

## 二十、核心概念

### 20.1 模板引擎选择

| 引擎 | 适用场景 | 特点 |
|------|---------|------|
| **docxtpl** | Word文档模板 | 基于Jinja2，保留Word格式，支持循环/条件 |
| **python-docx** | 程序化创建Word | 从零构建文档，精细控制格式 |

**推荐**：使用 **docxtpl** 进行模板驱动生成，业务人员可直接在Word中编辑模板。

### 20.2 Jinja2语法简介

```jinja2
{{ variable }}           # 变量输出
{{ variable | filter }}  # 带过滤器的变量
{% if condition %}       # 条件判断
{% for item in list %}   # 循环
{% endif %}
{% endfor %}
```

**docxtpl扩展语法**：

| 语法 | 用途 | 示例 |
|------|------|------|
| `{%p if %}` | 段落级条件 | `{%p if show_section %}内容{%p endif %}` |
| `{%tr for %}` | 表格行循环 | `{%tr for item in items %}{{ item.name }}{%tr endfor %}` |
| `{%tc for %}` | 表格列循环 | 较少使用 |
| `{{r var }}` | 富文本变量 | 支持动态样式 |

## 二十一、变量命名规范

### 21.1 命名原则

采用 **层级命名法**，使用点号分隔：

```
{{ 域.子域.字段 }}
```

**命名规则**：
1. 全部小写字母
2. 使用下划线连接多词
3. 避免使用中文变量名（值可以是中文）
4. 保持命名一致性

### 21.2 推荐域前缀

| 前缀 | 用途 | 示例 |
|------|------|------|
| `g` | 全局变量 | `g.company_name`, `g.report_no` |
| `info` | 基本信息 | `info.address`, `info.contact` |
| `data` | 数据内容 | `data.items`, `data.total` |
| `flags` | 布尔标志 | `flags.show_detail` |
| `notes` | 文字说明 | `notes.remarks` |

### 21.3 命名示例

**好的命名**：
```jinja2
{{ g.company_full_name }}
{{ info.registration_date }}
{{ data.invoice_items }}
{{ flags.has_attachment }}
```

**避免使用**：
```jinja2
{{ 公司名称 }}           # 中文变量名
{{ CompanyName }}        # 大写字母
{{ g.companyFullName }}  # 驼峰命名
{{ x }}                  # 无意义单字符
```

## 二十二、占位符替换规范

### 22.1 占位符类型

| 类型 | 原始格式 | 替换为 | 说明 |
|------|---------|--------|------|
| 文本占位符 | `××××××` 或 `XXXXXX` | `{{ notes.placeholder_描述 }}` | 需要填写的说明文字 |
| 数字占位符 | `【X】` 或 `【XX】` | `{{ data.number_field }}` | 需要填写的数字 |
| 日期占位符 | `【日期】` | `{{ g.date_field }}` | 需要填写的日期 |

### 22.2 占位符识别规则

**正则表达式**：

```python
# 文本占位符（连续4个及以上×或X）
PLACEHOLDER_PATTERN = re.compile(r"(?<![a-zA-Z_])[×X]{4,}(?![a-zA-Z_}])")

# 数字占位符
NUMBER_PLACEHOLDER_PATTERN = re.compile(r"【X+】")

# 通用占位符（【...】格式）
GENERIC_PLACEHOLDER_PATTERN = re.compile(r"【([^】]+)】")
```

### 22.3 描述标识生成规则

占位符应替换为有意义的变量名，生成规则：

**优先级1：从上下文提取**
```
"说明：××××××"     →  notes.placeholder_说明
"受限资金：××××××" →  notes.placeholder_受限资金
"金融资产的××××××" →  notes.placeholder_金融资产
```

**优先级2：使用序号标识**
```
无法提取时：notes.placeholder_001, notes.placeholder_002, ...
```

## 二十三、替换执行顺序

```
原始Word文档
    ↓
步骤1: 清理操作指引
    - 删除【注：...】等说明文字
    - 删除不需要出现在最终文档中的提示
    ↓
步骤2: 全局变量替换
    - 公司名称、编号、日期等固定字段
    - 页眉页脚中的固定文本
    ↓
步骤3: 占位符替换
    - ×××××× → {{ notes.placeholder_xxx }}
    - 【X】 → {{ data.xxx }}
    ↓
步骤4: 添加条件/循环结构（手动）
    - 根据需要添加 {%p if %} 条件
    - 根据需要添加 {%tr for %} 循环
    ↓
docxtpl模板
```

## 二十四、Run合并策略

### 24.1 问题背景

Word文档中，同一段落的文本可能被拆分到多个 **Run** 对象：

```xml
<w:p>
  <w:r><w:t>上海</w:t></w:r>
  <w:r><w:t>申能</w:t></w:r>
  <w:r><w:t>公司</w:t></w:r>
</w:p>
```

直接替换 `"上海申能公司"` 会失败，因为文本分散在多个Run中。

### 24.2 解决方案

```python
def merge_runs_and_replace(paragraph, replacements_func):
    """合并Run并执行替换"""
    runs = paragraph.runs
    if not runs:
        return False
    
    # 1. 合并所有Run文本
    full_text = "".join(run.text for run in runs)
    if not full_text:
        return False
    
    # 2. 执行替换
    new_text = replacements_func(full_text)
    
    if new_text == full_text:
        return False
    
    # 3. 写回第一个Run，清空其余
    runs[0].text = new_text
    for run in runs[1:]:
        run.text = ""
    
    return True
```

### 25.3 布尔值处理

Excel中布尔值存储为字符串 `"TRUE"`/`"FALSE"`，需转换为Python布尔值：

```python
def to_bool(value):
    if isinstance(value, str):
        return value.strip().upper() == 'TRUE'
    return bool(value)
```

## 二十七、自定义过滤器

### 27.1 常用过滤器

| 过滤器 | 功能 | 示例 |
|--------|------|------|
| `money` | 金额千分位 | `{{ 1234567.89 \| money }}` → "1,234,567.89" |
| `percent` | 百分比 | `{{ 0.1567 \| percent }}` → "15.67%" |
| `date` | 日期格式化 | `{{ g.date \| date }}` → "2025-01-15" |
| `default` | 默认值 | `{{ data.x \| default('-') }}` |

### 27.2 过滤器实现

```python
def format_money(value):
    """金额格式化"""
    if value is None or value == '':
        return ''
    try:
        num = float(value)
        if num == 0:
            return ''
        return f"{num:,.2f}"
    except (ValueError, TypeError):
        return str(value)

def format_percent(value):
    """百分比格式化"""
    if value is None or value == '':
        return ''
    try:
        return f"{float(value) * 100:.2f}%"
    except (ValueError, TypeError):
        return str(value)
```

## 三十、最佳实践

### 30.1 模板设计原则

1. **分离内容与格式**：样式定义在Word模板中，数据通过变量注入
2. **变量命名清晰**：使用有意义的变量名，避免缩写
3. **条件控制粒度**：段落级用 `{%p if %}`，行内用标准 `{% if %}`
4. **数据验证**：生成前校验必填字段和数据类型

### 30.2 常见问题

**Q1: 替换后格式丢失？**  
A: 确保使用Run合并策略，不要直接操作单个Run。

**Q2: Jinja2标签显示为纯文本？**  
A: 检查标签是否被Word拆分到多个Run中，使用合并策略解决。

**Q3: 动态表格格式错乱？**  
A: 确保 `{%tr for %}` 和 `{%tr endfor %}` 在同一行内，不要跨行。

**Q4: 条件判断不生效？**  
A: 检查布尔值是否正确转换，Excel中的 `"TRUE"` 需要转为 Python `True`。
