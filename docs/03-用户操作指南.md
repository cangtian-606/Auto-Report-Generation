## 二、Excel 数据填写

> 详细的格式规范见 [02-数据格式规范](./02-数据格式规范.md)，此处补充操作指引和常见模式。

### 2.1 键值对 Sheet

适用于单个值（公司名、日期、金额等）：

| 字段编码 | 值 |
|----------|-----|
| 公司名称 | 深圳市创新科技有限公司 |
| 注册资本 | 1000 |
| 成立日期 | 2018-06-15 |
| 有三免三减半 | TRUE |

### 2.2 表格 Sheet

适用于列表数据（股东、投资明细等）：

| 股东 | 认缴金额 | 认缴比例 |
|------|---------|---------|
| 张伟 | 600     | 0.60    |
| 李芳 | 250     | 0.25    |
| 王强 | 150     | 0.15    |

### 2.3 条件显示

某些内容只在特定条件下出现，通过布尔值控制：

- 填写 `TRUE` → 显示该块
- 填写 `FALSE` 或留空 → 隐藏该块
- 命名建议：以"有"或"是否"开头（如 `有三免三减半`、`有小微企业`）

## 三、命令执行

### 3.1 单文件生成

```bash
python -m src --data 数据.xlsx --template 模板.docx --output 输出.docx
```

简化：
```bash
python -m src -d 数据.xlsx -t 模板.docx -o 输出.docx
```

`--data` 支持 `.xlsx`、`.xls`、`.yaml`、`.yml`。

### 3.2 批量生成

```bash
python -m src --batch 数据目录/ --template 模板.docx --output-dir 输出目录/
```

简化：
```bash
python -m src -b 数据目录/ -t 模板.docx --output-dir 输出目录/
```

批量模式从目录中找所有数据文件，对每个执行渲染。

### 3.3 常用选项

| 选项 | 简写 | 说明 |
|------|:--:|------|
| `--data FILE` | `-d` | 数据文件 |
| `--template FILE` | `-t` | 模板文件 |
| `--output FILE` | `-o` | 输出文件 |
| `--batch DIR` | `-b` | 数据目录（批量模式） |
| `--output-dir DIR` | | 输出目录（批量模式） |
| `--strict` | | 严格模式：未定义变量时报错 |
| `--no-check` | | 跳过变量检查 |
| `--validate` | | 启用 Schema 验证 |
| `--strict-validate` | | 验证失败时报错退出 |
| `--schema FILE` | | Schema 文件 |
| `--check-syntax` | | 渲染前检查语法 |
| `--report-unused` | | 报告未使用的字段 |

详细参数表见 [06-附录](./06-附录-速查表.md#f-cli-参数速查)。

## 四、输出检查清单

生成文档后，逐一检查：

- [ ] 所有变量是否已替换（非空或符合预期）
- [ ] 条件显示是否正确（显示/隐藏符合预期）
- [ ] 循环数据行数是否正确
- [ ] 金额/百分比格式是否正确
- [ ] 页眉页脚是否正确
- [ ] 页面布局是否保持

## 五、错误处理

### 常见错误

| 错误信息 | 原因 | 解决方法 |
|---------|------|---------|
| 未定义变量 | 数据中缺少该字段 | 在 Excel 中添加该字段 |
| 找不到 Sheet | Sheet 名称不匹配 | 检查 Sheet 命名是否和模板一致 |
| 渲染失败 | 模板语法错误 | 联系 IT 检查模板 |
| 文件不存在 | 路径错误 | 检查文件名和目录 |

### 强制生成

使用 `--no-check` 跳过变量检查：

```bash
python -m src -d data.xlsx -t template.docx -o output.docx --no-check
```

## 六、模板获取

业务人员通常不需要制作模板，模板由 IT 部门或模板管理员提供。

**获取模板后**：
1. 确认模板版本
2. 了解所需填写的字段
3. 按本手册填写数据
4. 执行生成命令

**如需新模板或修改现有模板**，联系 IT 部门。
