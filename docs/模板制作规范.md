# 报告模板制作规范

## 面向 AI 的 Word 报告 → Jinja2 模板 + Excel 数据字典制作指南

---

## 一、概述

本文档定义一套标准化流程，指导 AI 将一份已填好的 Word 报告（样本）反向工程为：
1. **docxtpl 模板**（含 Jinja2 变量占位符的 .docx 文件）
2. **Excel 数据字典模板**（空的 .xlsx 文件，用于后续填写数据）

### 适用场景

| 场景 | 说明 |
|------|------|
| 新模板制作 | 从一份已填好的报告样本出发，制作可复用的模板 |
| 模板迁移 | 将旧格式固定文本报告转为参数化模板 |
| 批量生成准备 | 为批量文档生成准备模板 + 数据字典 |

---

## 二、核心原则

### 2.1 不变 vs 可变

| 类型 | 说明 | 处理方式 |
|------|------|---------|
| 固定文本 | 所有报告都相同的标题、提示、签字区等 | 保留原文 |
| 可变值 | 不同报告会变化的数据 | 替换为 `{{ 变量 }}` |
| 条件块 | 某些报告有、某些没有的段落/表格 | 用 `{%p if %}` 包裹 |
| 循环列表 | 行数不固定的表格 | 用 `{%tr for %}` 循环 |

### 2.2 两种数据类型

| 类型 | Excel Sheet 命名 | 模板引用方式 | 用途 |
|------|-----------------|-------------|------|
| 键值对 | 语义名（如 `全局信息`） | `{{ Sheet名.字段 }}` | 公司名、日期、编号、金额等单个值 |
| 循环表格 | 语义名（如 `股东出资`） | `{%tr for i in Sheet名 %}` | 股东列表、商品明细等列表数据 |

---

## 三、模板制作步骤

### 步骤1：分析报告结构

逐段阅读报告，将内容分为以下类别：

```
报告结构分析模板：

一、固定文本（保留）
   - 报告标题："投资项目概况报告"
   - 固定标签："公司名称："、"注册资本："等
   - 签字区、日期区

二、键值对数据（→ 独立 Sheet）
   - 公司名：深圳市创新科技有限公司  → 全局信息.公司名
   - 注册资本：1000                   → 基本情况.注册资本
   - 成立日期：2018-06-15            → 基本情况.成立日期

三、循环表格（→ 独立 Sheet）
   - 股东出资表（3行数据）            → 股东出资
   - 投资明细表（4行数据）            → 投资明细

四、条件块
   - 职工信息表（部分报告有）          → {%p if 基本情况.有职工信息 %}
```

### 步骤2：命名规范

#### Sheet 命名

```
格式：<语义名称>

直接使用中文语义名称，见名知义。

示例：
  全局信息    ← 封面/全局字段（公司名、日期、编号等）
  基本情况    ← 基本信息（注册资本、经营范围等）
  财务数据    ← 财务相关单个值
  股东出资    ← 股东出资列表
  投资明细    ← 投资明细列表
  商品清单    ← 商品/项目清单
```

#### 字段编码命名

```
Excel 中字段编码只写字段名，系统自动与 Sheet 名组合为 Sheet名.字段名：

Sheet: 全局信息
  字段编码: 公司名   → 模板变量: {{ 全局信息.公司名 }}
  字段编码: 日期     → 模板变量: {{ 全局信息.日期 }}

Sheet: 股东出资
  列标题: 股东       → 循环变量: {{ i.股东 }}
  列标题: 认缴金额   → 循环变量: {{ i.认缴金额 }}
```

#### 命名原则

1. 使用中文命名，见名知义
2. 同一报告内变量名保持唯一
3. 相似概念归入同一 Sheet（如注册信息统一放 `基本情况`）
4. 数值类变量考虑是否需要过滤器（`| num`、`| money`、`| percent`）
5. 布尔控制变量命名以"有"或"是否"开头（`有职工信息`、`有附件`）

### 步骤3：制作 Word 模板

#### 3.1 单值变量替换

将报告中的可变值替换为 `{{ variable }}` 占位符：

```
原文：
  公司名称：深圳市创新科技有限公司
  注册资本：1000万元

模板：
  公司名称：{{ 全局信息.公司名 }}
  注册资本：{{ 基本情况.注册资本 | num }}万元
```

#### 3.2 过滤器选择

根据值的类型选择合适的过滤器：

| 原始值类型 | 使用过滤器 | 效果示例 |
|-----------|-----------|---------|
| 金额数字 | `\| money` | 1234567.89 → 1,234,567.89 |
| 百分比小数 | `\| percent` | 0.25 → 25.00% |
| 普通数字 | `\| num` | 1234567 → 1,234,567 |
| 日期字符串 | `\| date` | 2025-01-15 → 2025年01月15日 |
| 整数 | `\| int` | 123.45 → 123 |
| 可空文本 | `\| default_dash` | 空 → "" |
| 可空文本 | `\| default('-')` | 空 → "-" |

> **重要**：空值和 0 值为空字符串""时，money/percent/num 过滤器输出为空字符串""。

#### 3.3 表格循环

将行数不固定的表格改为 `{%tr for %}` 循环：

```
原表格：
┌──────┬─────────┬─────────┐
│ 股东 │ 认缴金额 │ 认缴比例 │
├──────┼─────────┼─────────┤
│ 张伟 │ 600.00  │ 60.00%  │
│ 李芳 │ 250.00  │ 25.00%  │
│ 王强 │ 150.00  │ 15.00%  │
└──────┴─────────┴─────────┘

模板表格：
┌──────┬────────────────────┬──────────────────────┐
│ 股东 │ 认缴金额            │ 认缴比例              │
├──────┼────────────────────┼──────────────────────┤
│ {%tr for i in 股东出资 %}                     │
├──────┼────────────────────┼──────────────────────┤
│ {{ i.股东 }} │ {{ i.认缴金额 | money }} │ {{ i.认缴比例 | percent }} │
├──────┼────────────────────┼──────────────────────┤
│ {%tr endfor %}                                    │
└──────┴────────────────────┴──────────────────────┘
```

**关键规则**：
- `{%tr for %}` 行和 `{%tr endfor %}` 行各自占完整的一行
- 行数必须对齐表格列数
- 循环变量统一使用 `i`
- 合计行放在 `{%tr endfor %}` 之后

#### 3.4 条件块

将有条件出现的段落用 `{%p if %}` 包裹：

```
原报告（部分有）：
  职工信息：
  ┌──────┬─────────┬──────┐
  │ 序号 │ 姓名    │ 职务 │
  ├──────┼─────────┼──────┤
  │ 1    │ 张伟    │ 经理 │
  └──────┴─────────┴──────┘

模板：
  {%p if 基本情况.有职工信息 %}
  职工信息：
  ┌──────┬─────────────────────────────────┐
  │ 序号 │ 职务                            │
  ├──────┼─────────────────────────────────┤
  │ {%tr for i in 员工信息 %}          │
  ├──────┼─────────────────────────────────┤
  │ {{ i.序号 }} │ {{ i.职务 }}             │
  ├──────┼─────────────────────────────────┤
  │ {%tr endfor %}                          │
  └──────┴─────────────────────────────────┘
  {%p endif %}
```

#### 3.5 行内条件

段落内的条件判断使用标准 `{% if %}`：

```
{% if 折扣 > 0 %}
折扣：{{ 折扣 | money }}元
{% endif %}
```

### 步骤4：制作 Excel 数据字典

#### 4.1 整体结构

每个 Excel 文件包含一个或多个 Sheet，结构如下：

```
Excel 文件 (.xlsx)
├── Sheet: 全局信息    ← 键值对：公司名、日期、编号等
├── Sheet: 基本情况    ← 键值对：注册信息、条件标志等
├── Sheet: 股东出资    ← 表格：股东出资列表（含示例行）
├── Sheet: 员工信息    ← 表格：员工信息列表（含示例行）
└── Sheet: 投资明细    ← 表格：投资明细列表（含示例行）
```

#### 4.2 键值对条目(entry)格式

每个条目(entry)必须包含标题行（第一行），从第二行开始是数据：

| 字段编码 | 值 |
|----------|-----|
| 公司名 | |
| 日期 | |
| 报告编号 | |

**字段编码规则**：
- 只写字段名（系统自动与 Sheet 名组合为 Sheet名.字段名）
- 布尔值填写 `TRUE` 或 `FALSE`（不区分大小写）
- 数值直接填写数字
- 日期填写 `YYYY-MM-DD` 格式

**必须包含的标题行**：

| 列A | 列B |
|-----|-----|
| 字段编码 | 值 |

> 标题行 `字段编码 │ 值` 会被系统自动识别为键值对条目(entry)。

#### 4.3 表格条目(entry)格式

| 股东 | 认缴金额 | 认缴比例 | 实缴金额 | 实缴比例 |
|------|---------|---------|---------|---------|
| | | | | |
| | | | | |
| | | | | |

**格式规则**：
- 第一行是列标题（与模板中 `i.xxx` 的属性名一致）
- 后续行预留给数据填写
- 建议留 3 行示例空行方便填写
- 数值列设置单元格格式为"数值"
- 列标题不能为空，必须是中文且有意义的名称

#### 4.4 布尔控制字段

需要在某个键值对 Sheet 中定义控制变量：

| 字段编码 | 值 |
|----------|-----|
| 有职工信息 | FALSE |
| 有附件 | FALSE |
| 有备注 | TRUE |

> 默认为 `FALSE`（不显示），需要显示时改为 `TRUE`。

## 五、AI 执行清单

分析一份报告样本时，按以下顺序执行：

### 阶段1：结构分析

- [ ] 通读报告全文，列出所有段落和表格
- [ ] 区分固定文本和可变数据
- [ ] 识别表格的行数规律（固定行 → 直接替换；不固定行 → `{%tr for %}`）
- [ ] 识别条件出现的内容（某些报告有、某些没有 → `{%p if %}`）

### 阶段2：命名设计

- [ ] 为键值对数据分组建 Sheet（按语义分组，每组一个独立 Sheet）
- [ ] 为循环表格命名 Sheet（每个表格一个独立 Sheet）
- [ ] 字段编码使用中文，见名知义
- [ ] 为布尔控制变量命名（以"有"开头）

### 阶段3：模板制作

- [ ] 将每个可变值替换为 `{{ Sheet名.字段 }}`
- [ ] 根据值类型添加过滤器（money / percent / num / date / int / default_dash）
- [ ] 将不固定行数的表格改为 `{%tr for i in Sheet名 %}...{%tr endfor %}`
- [ ] 将条件块用 `{%p if %}...{%p endif %}` 包裹
- [ ] 表格合计行放在 `{%tr endfor %}` 之后

### 阶段4：数据字典制作

- [ ] 每个键值对 Sheet 建一个 Sheet，第一行 `字段编码 │ 值`
- [ ] 每个表格 Sheet 建一个 Sheet，第一行为列标题
- [ ] 布尔值默认为 `FALSE`
- [ ] 数值列保持空白，设置单元格格式为数值

### 阶段5：验证

- [ ] 每个 `{{ }}` 变量在 Excel 中都有对应的字段
- [ ] 每个条目(entry)名称与模板引用一致
- [ ] 表格列标题与模板中 `i.xxx` 的属性名一致
- [ ] 过滤器的使用与值类型匹配
- [ ] `{%p if %}` 和 `{%p endif %}` 成对出现
- [ ] `{%tr for %}` 和 `{%tr endfor %}` 成对出现

## 六、常见问题

### Q1: 合计行应该放在哪里？
A: 合计行放在 `{%tr endfor %}` 之后。如果合计值需要动态计算，有两种方案：
- 在 Excel 数据中新增一行"合计"，但会被循环一起渲染（不推荐）
- 将合计值作为独立的键值对变量放在合适 Sheet 中

### Q2: 一个报告中"不固定"和"有条件"的判断标准？
A: "不固定"指数据行数会变化（如股东人数可能是 1-10 人）。"有条件"指某些段落/表格在部分报告中完全不存在。

### Q3: 日期格式如何统一？
A: Excel 中统一使用 `YYYY-MM-DD` 格式，模板中可选配 `| date` 过滤器转为中文格式。

### Q4: 金额为0时模板应该显示什么？
A: 金额为 0 时 `| money` 过滤器输出为空字符串""。如需显示 "0.00"，请在不使用 `| money` 过滤器的情况下直接渲染数值。

### Q5: 同一份报告需要多语言版本怎么办？
A: 固定文本需创建多份不同语言的模板。可变数据可以复用同一份 Excel 数据文件。

## 七、变量语法速查

| 语法 | 说明 | 示例 |
|------|------|------|
| `{{ var }}` | 输出变量值 | `{{ 全局信息.公司名 }}` |
| `{{ var \| filter }}` | 带过滤器输出 | `{{ 基本情况.注册资本 \| num }}` |
| `{%p if cond %}...{%p endif %}` | 段落级条件 | `{%p if 基本情况.有职工信息 %}` |
| `{% if cond %}...{% endif %}` | 行内条件 | `{% if 折扣 > 0 %}` |
| `{%tr for i in xxx %}...{%tr endfor %}` | 表格行循环 | `{%tr for i in 股东出资 %}` |

| 过滤器 | 功能 | 示例输出 |
|--------|------|---------|
| `\| money` | 千分位金额 | 1,234,567.89 |
| `\| percent` | 百分比（自动乘100） | 25.00% |
| `\| num` | 千分位数字 | 1,234,567 |
| `\| date` | 日期格式化 | 2025年01月15日 |
| `\| int` | 整数 | 123 |
| `\| default_dash` | 空值转字符串 |
| `\| default('x')` | 空值显示 "x" | 自定义默认值 |

## 八、循环嵌套

### 8.1 块级循环（`{% for %}`）

当需要复制一整段内容（含多个段落 + 表格）时，使用块级循环：

```
{% for company in 项目公司 %}

  {{ company.公司简称 }} 尽职调查报告

  公司名称：{{ company.公司名称 }}

  股东出资：
  ┌──────┬────────┬──────┐
  │ 股东 │ 认缴额 │ 比例 │
  ├──────┼────────┼──────┤
  {%tr for i in company.股东出资 %}
  │ {{ i.股东 }} │ {{ i.认缴额 | money }} │ {{ i.比例 | percent }} │
  {%tr endfor %}
  └──────┴────────┴──────┘

{% endfor %}
```

**规则**：`{% for %}` 和 `{% endfor %}` 各占独立一段，之间的所有内容被整体复制 N 次（N = 数据行数）。

### 8.2 嵌套语法速查

| 语法 | 层级 | 能包裹什么 |
|------|------|----------|
| `{% for %} ... {% endfor %}` | 段落块级 | 多段落 + 多表格 |
| `{%tr for %} ... {%tr endfor %}` | 表格行级 | 仅表格行 |
| `{%p if %} ... {%p endif %}` | 段落级 | 仅所在段落 |

### 8.3 Excel 嵌套数据填写

**父 Sheet**：`项目公司`（含有子表的父列表）

| 公司简称 | 公司名称 | 信用代码 |
|----------|---------|---------|
| 重庆晟和泰 | 重庆晟和泰新能源科技有限公司 | 91500227MA5U7XXX |
| 安徽富军 | 安徽富军新能源有限公司 | 91341525MA2TXXXX |

**子 Sheet**：`项目公司.股东出资`（点号分隔表母子关系）

| _parent_公司简称 | 股东 | 认缴额 | 比例 |
|-----------------|------|--------|------|
| 重庆晟和泰 | 张伟 | 600 | 0.60 |
| 重庆晟和泰 | 李芳 | 250 | 0.25 |
| 安徽富军 | 王强 | 500 | 0.50 |

> **关键规则**：
> - 子 Sheet 名 = `父Sheet名.子名`（点号分隔）
> - 子 Sheet 第一列必须以 `_parent_` 开头，后面跟父 Sheet 的匹配列名
> - `_parent_公司简称` 的值必须与父 Sheet `公司简称` 列的某一行精确一致
> - 子行找不到父行 → 报错，父行无子数据 → 报错

### 8.4 三种嵌套关系

| 父类型 | 子类型 | Sheet命名 | 关联 | 用途 |
|--------|--------|----------|------|------|
| 表格 | 表格 | `父.子` | `_parent_`列匹配 | 多公司各有股东表 |
| 键值对 | 键值对 | `父.子` | 自动挂载 | 单公司信息分Sheet填写 |
| 键值对 | 表格 | `父.子`（表格） | 自动挂载 | 单公司但有列表子数据 |

### 8.5 YAML 备选输入

开发阶段可直接用 YAML 文件替代 Excel，结构直观且无需 `_parent_` 列：

```yaml
  项目公司:
    - 公司简称: 重庆晟和泰
      公司名称: 重庆晟和泰新能源科技有限公司
      信用代码: 91500227MA5U7XXX
      股东出资:
        - 股东: 张伟
          认缴额: 600
          比例: 0.60
        - 股东: 李芳
          认缴额: 250
          比例: 0.25
    - 公司简称: 安徽富军
      公司名称: 安徽富军新能源有限公司
      信用代码: 91341525MA2TXXXX
      股东出资:
        - 股东: 王强
          认缴额: 500
          比例: 0.50
```

```python
from src.yaml_reader import YamlDataReader
reader = YamlDataReader('data.yaml')
ctx = reader.read_context()  # 直接是可用的 context
```

> **注意**：YAML 数据输入同样会经过 DataMapper 的值转换处理（None→''，TRUE/FALSE→bool），与 Excel 路径行为一致。例如 YAML 中的 `null` 会被转为空字符串，`true`/`false` 会被转为 Python 布尔值。
