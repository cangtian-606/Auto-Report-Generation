# Domain Docs

工程技能在探索代码库时应如何读取本仓库的领域文档。

## 探索前先阅读

- **`CONTEXT.md`** — 仓库根目录下的领域上下文文件
- **`docs/adr/`** — 架构决策记录，先读与你即将工作的领域相关的 ADR

如果这些文件不存在，**静默跳过**。不要提示它们缺失，也不要主动建议创建。`/grill-with-docs` 技能会在术语或决策实际确定时按需创建。

## 文件结构

Single-context 仓库：

```
/
├── CONTEXT.md
├── docs/adr/
│   ├── 0001-xxx.md
│   └── 0002-xxx.md
└── src/
```

## 使用术语表中的词汇

当你的输出中需要命名领域概念（issue 标题、重构提案、假设、测试名称），使用 `CONTEXT.md` 中定义的术语。不要漂移到术语表明确避免的同义词。

如果你需要的概念尚未在术语表中定义，这是一个信号——要么你在创造项目不使用的语言（应重新考虑），要么确实存在空缺（可以记下供 `/grill-with-docs` 补充）。

## 标记 ADR 冲突

如果你的输出与现有 ADR 矛盾，应显式指出而非静默覆盖：

> _与 ADR-0007 矛盾 — 但值得重新讨论，因为……_
