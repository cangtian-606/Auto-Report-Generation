# 模板制作规范

> 面向 AI / 模板制作者：将 Word 报告样本反向工程为 docxtpl 模板 + Excel 数据字典

阅读本文前，请先了解：[02-数据格式规范](./02-数据格式规范.md)

---

## 一、概述

本文档定义标准化流程，指导将一份已填好的 Word 报告样本反向工程为：

1. **docxtpl 模板**（含 Jinja2 变量占位符的 `.docx`）
2. **Excel 数据字典**（空的 `.xlsx`，用于后续填写数据）

### 适用场景

| 场景 | 说明 |
|------|------|
| 新模板制作 | 从已填好的报告样本出发，制作可复用模板 |
| 模板迁移 | 将旧格式固定文本报告转为参数化模板 |
| 批量生成准备 | 为批量文档生成准备模板和数据字典 |

### 模板标签体系

docxtpl 中所有标签按前缀分为三个作用域层级：

| 层级 | 前缀 | 能包裹什么 | 典型标签 |
|------|:--:|------|------|
| **块级** | 无前缀 | 多段落 + 表格 + 图片等任意内容 | `{% for %}`, `{% if %}` |
| **段落区** | `{%p %}` | 多个纯文字段落，**不能含表格** | `{%p if %}` |
| **行内** | `{%r %}` | 段落中的一段文字（单个 run） | `{%r if %}` |

此外还有表格专用标签：

| 标签前缀 | 用途 | 示例 |
|----------|------|------|
| `{%tr for %}` | **表格行循环**（复制表格行） | `{%tr for i in 历史沿革.股权结构 %}` |
| `{%tc for %}` | **表格列循环**（横向展开列） | `{%tc for i in 项目基本情况 %}` |

> **关键区分**：`{% if %}` vs `{%p if %}` vs `{%r if %}` —— 块级(`{% %}`)可包裹含表格的大区块，段落区(`{%p %}`)只能控制纯文字段落，行内(`{%r %}`)只控制一行中的几个字。选择规则：有表格→`{% if %}`，纯文字→`{%p if %}`，几个字→`{%r if %}`。

## 二、核心原则

### 2.1 不变 vs 可变

| 类型 | 说明 | 处理方式 |
|------|------|---------|
| 固定文本 | 所有报告都相同的标题、提示、签字区 | 保留原文 |
| 可变值 | 不同报告会变化的数据 | 替换为 `{{ 变量 }}` |
| 条件块（含表格） | 某些报告有、某些没有的表格区块 | 用 `{% if %}` 包裹 |
| 条件段（纯文字） | 某些报告有、某些没有的段落 | 用 `{%p if %}` 包裹 |
| 行内条件 | 一句话中几个字是否显示 | 用 `{%r if %}` 包裹 |
| 行循环列表 | 行数不固定的表格 | 用 `{%tr for %}` 循环 |
| 段落循环列表 | 个数不固定的段落块 | 用 `{% for %}` 循环 |
| 列循环列表 | 列数不固定的表格 | 用 `{%tc for %}` 循环 |
| 嵌套列表 | 列表项内含子表 | 用 `{% for %}` + `{%tr for %}` 嵌套 |

## 三、模板制作步骤

### 步骤 1：分析报告结构

逐段阅读报告，分类：

```
一、固定文本（保留）
   - 报告标题、固定标签、签字区

二、键值对数据（→ 独立 Sheet）
   - 公司名称 → 全局.公司名称
   - 注册资本 → 历史沿革.注册资本

三、行循环表格（→ 独立 Sheet）
   - 股权结构表 → 历史沿革.股权结构
   - 释义表 → 释义

四、段落循环（→ 列表型 Sheet）
   - 多个项目的基本情况 → 项目基本情况
   - 多个EPC合同信息 → EPC合同

五、嵌套列表（→ 父列表 + 子表）
   - EPC合同中的金额明细 → EPC合同.epc金额

六、条件块
   - 三免三减半政策（部分报告有）→ {%p if 税率政策.有三免三减半 %}
```

### 步骤 2：命名设计

#### Sheet 命名

使用中文语义名称，见名知义：

```
全局        ← 封面/全局字段
释义        ← 全称-简称对照表
历史沿革   ← 公司基本信息
项目基本情况 ← 多个项目核心参数（段落循环用）
EPC合同    ← EPC工程合同列表
EMC合同    ← EMC能源管理合同列表
购售电合同  ← 购售电合同列表
税率政策   ← 税率及优惠条件
```

嵌套子表用 `.` 连接父表名：

```
历史沿革.股权结构   ← 历史沿革的子表格
历史沿革.公司设立   ← 历史沿革的子表格
EPC合同.epc金额    ← EPC合同的子表格（金额明细）
```

#### 字段命名原则

1. 使用中文，见名知义
2. 同一报告内变量名保持唯一
3. 相似概念归入同一 Sheet
4. 布尔控制字段以"有"或"是否"开头（`有三免三减半`、`有小微企业`、`有流转税减免`）

### 步骤 3：制作 Word 模板

#### 3.1 单值变量替换

```
原文：
  公司名称：深圳市创新科技有限公司
  注册资本：1000万元

模板：
  公司名称：{{ 全局.公司名称 }}
  注册资本：{{ 历史沿革.注册资本 | num }}万元
```

#### 3.2 过滤器选择

> 完整过滤器列表见 [02-数据格式规范](./02-数据格式规范.md#七过滤器速查)

简表：

| 值类型 | 使用 | 示例输入 → 输出 |
|--------|------|----------------|
| 金额 | `\| num` | 1234567.89 → 1,234,567.89 |
| 百分比小数 | `\| percent` | 0.25 → 25.00% |
| 普通数字 | `\| num(0)` | 1234567 → 1,234,567 |
| 日期 | `\| date` | 2025-01-15 → 2025年01月15日 |
| 整数 | `\| int` | 123.45 → 123 |

#### 3.3 行循环表格 `{%tr for %}`

将行数不固定的表格改为 `{%tr for %}` 循环：

```
模板表格：
┌──────────┬──────────────────┬────────────────────┐
│ 股东名称 │ 认缴出资额         │ 认缴比例             │
├──────────┼──────────────────┼────────────────────┤
│ {%tr for i in 历史沿革.股权结构 %}                 │
├──────────┼──────────────────┼────────────────────┤
│ {{ i.股东名称 }} │ {{ i.认缴出资额\|money }} │ {{ i.认缴比例\|percent }} │
├──────────┼──────────────────┼────────────────────┤
│ {%tr endfor %}                                     │
└──────────┴──────────────────┴────────────────────┘
```

**关键规则**：
- `{%tr for %}` 和 `{%tr endfor %}` 各占完整一行
- 行数必须对齐表格列数
- 循环变量统一使用 `i`
- 合计行放在 `{%tr endfor %}` 之后

#### 3.4 嵌套循环表格（列表项内的子表）

当列表的每一项包含子表格时，先外层 `{% for %}` 再内层 `{%tr for %}`：

```
{% for i in EPC合同 %}

EPC承包单位：{{ i.承包单位 }}
工程内容及规模：{{ i.工程内容及规模 }}

合同金额明细：
┌────────┬──────────┬─────────┬──────────────┐
│ 序号   │ 合同分项  │ 进项税率 │ 含税金额      │
├────────┼──────────┼─────────┼──────────────┤
│ {%tr for k in i.epc金额 %}                    │
├────────┼──────────┼─────────┼──────────────┤
│ {{ k.序号\|int }} │ {{ k.合同分项 }} │ {{ k.税率\|percent }} │ {{ k.分项金额\|num }} │
├────────┼──────────┼─────────┼──────────────┤
│ {%tr endfor %}                                │
├────────┼──────────┼─────────┼──────────────┤
│ 合计   │          │         │ {{ i.合同金额\|num }} │
└────────┴──────────┴─────────┴──────────────┘

{% endfor %}
```

> **提示**：系统会自动将子表扁平化为顶级路径 `EPC合同.epc金额`，因此在不需要父级上下文时也可独立使用 `{%tr for k in EPC合同.epc金额 %}`。

#### 3.5 段落循环 `{% for %}`

当需要循环的不是表格行而是整个段落块时：

```
{% for i in 项目基本情况 %}

{{ i.项目名称 }}

项目地址：{{ i.项目地址 }}
装机容量：{{ i.项目装机量 }}
运行模式：{{ i.项目模式 }}
含税总价：{{ i.含税总价 | num }}万元

{% endfor %}
```

**规则**：`{% for %}` 和 `{% endfor %}` 各占独立一个段落，之间内容被整体复制 N 次。可包含多段落、表格等任意内容。

#### 3.6 条件控制

根据要控制的内容选择正确的标签前缀：

| 场景 | 标签 | 写法要点 |
|------|------|---------|
| 含表格的区块 | `{% if %}` | `{% if %}` 和 `{% endif %}` 各占独立段落 |
| 纯文字段落（多段） | `{%p if %}` | `{%p if %}` 和 `{%p endif %}` 各占独立段落（回车），被包裹的段落条件显隐 |
| 一行中几个字 | `{%r if %}` | 在同一段落内使用 |

**段落区条件示例**：

```
{%p if 税率政策.有三免三减半 %}
根据相关税收优惠政策，本项目享受三免三减半企业所得税优惠。
{%p endif %}
```

> `{%p if %}` 和 `{%p endif %}` 所在段落会被删除，只有中间的内容条件显示。可以包裹多个段落（但**不能含表格**）。

**行内条件示例**：

```
合同金额：{{ 合同金额 | num }}{%r if 折扣 > 0 %}（折扣后）{%r endif %}
```

> `{%r if %}` 只能控制一行中的几个字，不产生新段落，保留当前段落的首行缩进等样式。

## 四、循环嵌套

### 4.1 列循环 `{%tc for %}`

当表格列数不固定（如按项目动态展开列），使用 `{%tc for %}` 横向复制单元格。

**典型场景**：多个项目并列对比，每个项目占一列。

**数据**：

```yaml
项目基本情况:
  - 项目名称: "深圳分布式光伏"
    备案文件: "已完成"
    接入电网审批: "已批复"
    环评文件: "已备案"
  - 项目名称: "广州分布式光伏"
    备案文件: "已完成"
    接入电网审批: "办理中"
    环评文件: "已备案"
```

**模板**（表头行声明 `{%tc for %}`，后续数据行自动继承）：

```
┌──────┬────────────────────────────────────────┐
│ 项目 │ {%tc for i in 项目基本情况 %}              │
│ 名称 │ {{ i.项目名称 }}                          │
│      │ {%tc endfor %}                           │
├──────┼────────────────────────────────────────┤
│ 备案 │ {%tc for i in 项目基本情况 %}              │
│ 文件 │ {{ i.备案文件 }}                          │
│      │ {%tc endfor %}                           │
├──────┼────────────────────────────────────────┤
│ 接入 │ {%tc for i in 项目基本情况 %}              │
│ 电网 │ {{ i.接入电网审批 }}                      │
│      │ {%tc endfor %}                           │
└──────┴────────────────────────────────────────┘
```

**效果**：

| 项目名称 | 深圳分布式光伏 | 广州分布式光伏 |
|------|-----------|-----------|
| 备案文件 | 已完成 | 已完成 |
| 接入电网 | 已批复 | 办理中 |

**关键规则**：
- `{%tc for %}` 和 `{%tc endfor %}` 各占独立一格
- 中间的 `{{ i.属性 }}` 占一格——它是被动态复制的"模板列"
- N 个元素 → 最终生成 N 列（替换掉 tc 标签格 + 模板格）
- **首行声明，整表继承**：首行（表头行）声明 `{%tc for %}` 后，系统自动将后续行的相同列位置包裹为隐式 `{%tc for %}` 块，后续行无需重复书写
- 迭代数据为对象列表，使用 `i.字段名` 点号访问属性
- 可以与 `{%tr for %}` 嵌套使用实现行列双动态

> **自动继承机制**：表格中只要有一行写了 `{%tc for %}`，系统（`TcInheritancePreprocessor`）会在渲染前自动将后续行的对应列位置包裹为列循环标签。因此实际制作模板时，通常只需在表头行书写完整的 `{%tc for %}` 块。

### 4.2 块级循环

当需要复制一整段内容（含多段落 + 表格），与 3.5 节段落循环相同，但可在其中嵌套表格：

```
{% for company in 项目公司 %}

{{ company.公司简称 }} 尽职调查报告

股权结构：
┌──────┬────────┬──────┐
│ 股东 │ 认缴额 │ 比例 │
├──────┼────────┼──────┤
{%tr for i in company.股东出资 %}
│ {{ i.股东 }} │ {{ i.认缴额 | num }} │ {{ i.比例 | percent }} │
{%tr endfor %}
└──────┴────────┴──────┘

{% endfor %}
```

**规则**：`{% for %}` 和 `{% endfor %}` 各占独立一段，之间内容被整体复制 N 次。

### 4.3 嵌套语法层级

| 语法 | 层级 | 能包裹什么 |
|------|------|----------|
| `{% for %} ... {% endfor %}` | **块级** | 多段落 + 多表格 + 图片等任意内容 |
| `{% if %} ... {% endif %}` | **块级** | 多段落 + 多表格 + 图片等任意内容 |
| `{%p if %} ... {%p endif %}` | **段落区** | 多个纯文字段落，不能含表格 |
| `{%tc for %} ... {%tc endfor %}` | 表格列级 | 同行单元格 |
| `{%tr for %} ... {%tr endfor %}` | 表格行级 | 仅表格行 |
| `{%r if %} ... {%r endif %}` | **行内** | 段落中的一段文字 |

**嵌套规则**（从外到内）：
```
块级（{% for %}/{% if %}）
  └── 段落区（{%p if %}）
        └── 行内（{%r if %}）
              └── 变量（{{ }}）
表格（{%tr for %}/{%tc for %}）
  └── 行内（{%r if %}）
        └── 变量（{{ }}）
```

### 4.4 嵌套数据填写

> 详见 [02-数据格式规范 - 嵌套数据](./02-数据格式规范.md#六嵌套数据)

**关键规则**：
- 子 Sheet 名 = `父Sheet名.子名`（如 `EPC合同.epc金额`）
- 列表型父表下的子表首列必须以 `_parent_` 开头
- 键值对型父表下的子表无需 `_parent_` 列
- 子行找不到父行 → 报错，父行无子数据 → 报错
- 系统自动扁平化子表为顶级路径，可直接用顶级路径访问

## 五、AI 执行清单

分析一份报告样本时，按以下顺序执行：

### 阶段 1：结构分析
- [ ] 通读全文，列出所有段落和表格
- [ ] 区分固定文本和可变数据
- [ ] 识别不固定行数表格 → `{%tr for %}`
- [ ] 识别不固定数量的段落块 → `{% for %}`
- [ ] 识别列表项内含子表的场景 → `{% for %}` + `{%tr for %}` 嵌套
- [ ] 识别条件内容 → 含表格用 `{% if %}`，纯文字用 `{%p if %}`
- [ ] 识别列数不固定的表格 → `{%tc for %}`

### 阶段 2：命名设计
- [ ] 为键值对数据分组建 Sheet
- [ ] 为循环表格命名 Sheet
- [ ] 为段落循环列表命名 Sheet
- [ ] 为嵌套子表用点号命名（`父表.子表`）
- [ ] 字段编码使用中文
- [ ] 布尔变量以"有"或"是否"开头

### 阶段 3：模板制作
- [ ] 可变值 → `{{ Sheet.字段 }}`
- [ ] 添加过滤器（num / percent / date / int）
- [ ] 不固定行数表格 → `{%tr for %}...{%tr endfor %}`
- [ ] 不固定段落块 → `{% for %}...{% endfor %}`
- [ ] 列表内子表 → `{%tr for k in i.子表 %}`
- [ ] 条件段（纯文字） → `{%p if %}...{%p endif %}`
- [ ] 条件块（含表格） → `{% if %}...{% endif %}`
- [ ] 行内条件 → `{%r if %}...{%r endif %}`
- [ ] 合计行放在 `{%tr endfor %}` 之后

### 阶段 4：数据字典制作
- [ ] 每个键值对 Sheet：第一行 `字段编码 │ 值`
- [ ] 每个表格 Sheet（含子表）：第一行为列标题
- [ ] 列表型父表的子表首列以 `_parent_` 开头
- [ ] 布尔值默认 `FALSE`
- [ ] 数值列设置单元格格式为数值

### 阶段 5：验证
- [ ] 每个 `{{ }}` 在 Excel 中都有对应字段
- [ ] Sheet 名与模板引用一致
- [ ] `{%p if %}` / `{%p endif %}` 成对
- [ ] `{%tr for %}` / `{%tr endfor %}` 成对
- [ ] `{% for %}` / `{% endfor %}` 成对
- [ ] `{%tc for %}` / `{%tc endfor %}` 成对

## 六、常见问题

**Q: 什么情况下用 `{% for %}`，什么情况下用 `{%tr for %}`？**
A: `{%tr for %}` 仅用于表格行循环（必须在一个 Word 表格内）。`{% for %}` 用于块级循环，可包含多段落、表格、图片等任意内容。列表项内包含子表格时，外层 `{% for %}` + 内层 `{%tr for %}`。

**Q: `{% if %}` 和 `{%p if %}` 有什么区别？**
A: `{% if %}`（块级）可包裹含表格的区块；`{%p if %}`（段落区）只能控制纯文字段落。规则：有表格→`{% if %}`，纯文字→`{%p if %}`, 几个字→`{%r if %}`。

**Q: 嵌套子表如何访问？**
A: 两种方式：1) 在父循环内用 `{%tr for k in i.epc金额 %}`（属性路径）；2) 系统自动扁平化为顶级路径 `{%tr for k in EPC合同.epc金额 %}`。

**Q: 合计行放在哪里？**
A: 放在 `{%tr endfor %}` 之后。如需动态合计，将合计值作为独立键值对变量。

**Q: 什么算"不固定"？什么算"有条件"？**
A: "不固定"指数据条数会变化（如 1-10 个项目）。"有条件"指某些段落仅在部分报告中存在。

**Q: 日期格式如何统一？**
A: Excel 中统一用 `YYYY-MM-DD`，模板中可选 `| date` 转中文格式。

**Q: 金额为 0 时显示什么？**
A: 金额为 0 时 `| num` 输出空。如需显示 "0.00"，直接用 `{{ 变量 }}` 不加过滤器。

**Q: `{%tc for %}` 自动继承是什么意思？**
A: 模板中只需在第一行书写完整的 `{%tc for %}` 块，后续行的相同列位置会被系统（`TcInheritancePreprocessor`）自动注入列循环标签，无需手动重复书写。
