# 模板制作规范

> 面向 AI / 模板制作者：将 Word 报告样本反向工程为 docxtpl 模板 + Excel 数据字典

阅读本文前，请先了解：[02-数据格式规范](./02-数据格式规范.md)

---

## 一、概述

本文档定义标准化流程，指导将一份已填好的 Word 报告样本反向工程为：

1. **docxtpl 模板**（含 Jinja2 变量占位符的 `.docx`）
2. **Excel 数据字典**（空的 `.xlsx`，用于后续填写数据）

### 适用场景

| 场景 | 说明 |
|------|------|
| 新模板制作 | 从已填好的报告样本出发，制作可复用模板 |
| 模板迁移 | 将旧格式固定文本报告转为参数化模板 |
| 批量生成准备 | 为批量文档生成准备模板和数据字典 |

---

## 二、核心原则

### 2.1 不变 vs 可变

| 类型 | 说明 | 处理方式 |
|------|------|---------|
| 固定文本 | 所有报告都相同的标题、提示、签字区 | 保留原文 |
| 可变值 | 不同报告会变化的数据 | 替换为 `{{ 变量 }}` |
| 条件块 | 某些报告有、某些没有的段落/表格 | 用 `{%p if %}` 包裹 |
| 循环列表 | 行数不固定的表格 | 用 `{%tr for %}` 循环 |

## 三、模板制作步骤

### 步骤 1：分析报告结构

逐段阅读报告，分类：

```
一、固定文本（保留）
   - 报告标题、固定标签、签字区

二、键值对数据（→ 独立 Sheet）
   - 公司名 → 全局信息.公司名
   - 注册资本 → 基本情况.注册资本

三、循环表格（→ 独立 Sheet）
   - 股东出资表 → 股东出资

四、条件块
   - 职工信息（部分报告有）→ {%p if 基本情况.有职工信息 %}
```

### 步骤 2：命名设计

#### Sheet 命名

使用中文语义名称，见名知义：

```
全局信息  ← 封面/全局字段
基本情况  ← 基本信息
财务数据  ← 财务相关值
股东出资  ← 股东列表
投资明细  ← 投资明细列表
```

#### 字段命名原则

1. 使用中文，见名知义
2. 同一报告内变量名唯一
3. 相似概念归入同一 Sheet
4. 布尔控制字段以"有"或"是否"开头（`有职工信息`、`有附件`）

### 步骤 3：制作 Word 模板

#### 3.1 单值变量替换

```
原文：
  公司名称：深圳市创新科技有限公司
  注册资本：1000万元

模板：
  公司名称：{{ 全局信息.公司名 }}
  注册资本：{{ 基本情况.注册资本 | num }}万元
```

#### 3.2 过滤器选择

> 完整过滤器列表见 [02-数据格式规范](./02-数据格式规范.md#七过滤器速查)

简表：

| 值类型 | 使用 | 示例输入 → 输出 |
|--------|------|----------------|
| 金额 | `\| money` | 1234567.89 → 1,234,567.89 |
| 百分比小数 | `\| percent` | 0.25 → 25.00% |
| 普通数字 | `\| num` | 1234567 → 1,234,567 |
| 日期 | `\| date` | 2025-01-15 → 2025年01月15日 |
| 整数 | `\| int` | 123.45 → 123 |

#### 3.3 表格循环

将行数不固定的表格改为 `{%tr for %}` 循环：

```
模板表格：
┌──────┬────────────────────┬──────────────────────┐
│ 股东 │ 认缴金额            │ 认缴比例              │
├──────┼────────────────────┼──────────────────────┤
│ {%tr for i in 股东出资 %}                     │
├──────┼────────────────────┼──────────────────────┤
│ {{ i.股东 }} │ {{ i.认缴金额 | money }} │ {{ i.认缴比例 | percent }} │
├──────┼────────────────────┼──────────────────────┤
│ {%tr endfor %}                                    │
└──────┴────────────────────┴──────────────────────┘
```

**关键规则**：
- `{%tr for %}` 和 `{%tr endfor %}` 各占完整一行
- 行数必须对齐表格列数
- 循环变量统一使用 `i`
- 合计行放在 `{%tr endfor %}` 之后

#### 3.4 条件块

将有条件出现的段落用 `{%p if %}` 包裹：

```
{%p if 基本情况.有职工信息 %}
职工人数：{{ 基本情况.职工人数 }}人
{%p endif %}
```

#### 3.5 行内条件

```
合同金额：{{ 合同金额 | money }}{%r if 折扣 > 0 %}（折扣后）{%r endif %}
```

## 四、循环嵌套

### 4.1 列循环 `{%tc for %}`

当表格列数不固定（如按月份动态展开），使用 `{%tc for %}` 横向复制单元格：

**数据**：

```yaml
months: ["1月", "2月", "3月"]
data_产品A: {"1月": 100, "2月": 200, "3月": 150}
total_A: 450
```

**模板**（表头 + 数据行各含 `{%tc for %}`）：

```
┌──────┬───────────────────────────────────────┬──────────┐
│ 产品 │ {%tc for month in months %}             │ 合计     │
│      │ {{ month }}                           │          │
│      │ {%tc endfor %}                        │          │
├──────┼───────────────────────────────────────┼──────────┤
│ 产品A│ {%tc for month in months %}             │ {{ total_A | money }} │
│      │ {{ data_产品A[month] | money }}         │          │
│      │ {%tc endfor %}                        │          │
└──────┴───────────────────────────────────────┴──────────┘
```

**效果**：

| 产品 | 1月 | 2月 | 3月 | 合计 |
|------|-----|-----|-----|------|
| 产品A | 100.00 | 200.00 | 150.00 | 450.00 |

**关键规则**：
- `{%tc for %}` 和 `{%tc endfor %}` 各占独立一格
- 中间的 `{{ month }}` 占一格——它是被动态复制的"模板列"
- `months` 有 3 个元素 → 最终生成 3 列（替换掉 tc 标签格 + 模板格）
- 表头和每行数据都需要独立的 `{%tc for %}` 块
- 可以与 `{%tr for %}` 嵌套使用实现行列双动态
- 可与项目过滤器（`| money`）组合

> **注意**：`{%tc for %}` 只能在同一行内横向展开，不能跨行。如需多行动态数据配合动态列，每行各自写 `{%tc for %}`。
>
> **优先使用转置方案**：如果列名固定、仅行数变化，建议将数据转置为标准 `{%tr for %}` 模式。`{%tc for %}` 仅在列确实不可预测时使用。

### 4.2 块级循环

当需要根据列表数据重复生成一段或多段内容时使用。

**简单场景：段落列表**

数据中有一个项目公司列表，需要为每个公司生成一段描述：

```yaml
项目基本情况:
  项目公司:
    - 项目名称: 重庆晟和泰光伏电站
      项目地址: 重庆市巫山县
      项目装机量: 50MW
      项目模式: 全额上网
    - 项目名称: 安徽富军风电场
      项目地址: 安徽省六安市
      项目装机量: 100MW
      项目模式: 自发自用
```

**模板**：

```
{% for i in 项目基本情况.项目公司 %}
{{ i.项目名称 }}位于{{ i.项目地址 }}，实际装机容量为{{ i.项目装机量 }}，采用"{{ i.项目模式 }}"运行方式。
{% endfor %}
```

**效果**：

```
重庆晟和泰光伏电站位于重庆市巫山县，实际装机容量为50MW，采用"全额上网"运行方式。
安徽富军风电场位于安徽省六安市，实际装机容量为100MW，采用"自发自用"运行方式。
```

**复杂场景：嵌套段落 + 表格**

```
{% for company in 项目公司 %}

{{ company.公司简称 }} 尽职调查报告

股东出资：
┌──────┬────────┬──────┐
│ 股东 │ 认缴额 │ 比例 │
├──────┼────────┼──────┤
{%tr for i in company.股东出资 %}
│ {{ i.股东 }} │ {{ i.认缴额 | money }} │ {{ i.比例 | percent }} │
{%tr endfor %}
└──────┴────────┴──────┘

{% endfor %}
```

**规则**：`{% for %}` 和 `{% endfor %}` 各占独立一段，之间内容被整体复制 N 次。

### 4.3 嵌套语法层级

| 语法 | 层级 | 能包裹什么 |
|------|------|----------|
| `{% for %} ... {% endfor %}` | 段落块级 | 多段落 + 多表格 |
| `{%tc for %} ... {%tc endfor %}` | 表格列级 | 同行单元格 |
| `{%tr for %} ... {%tr endfor %}` | 表格行级 | 仅表格行 |
| `{%p if %} ... {%p endif %}` | 段落级 | 仅所在段落 |

### 4.4 嵌套数据填写

> 详见 [02-数据格式规范 - 嵌套数据](./02-数据格式规范.md#六嵌套数据)

**关键规则**：
- 子 Sheet 名 = `父Sheet名.子名`
- 子 Sheet 第一列必须以 `_parent_` 开头
- 子行找不到父行 → 报错，父行无子数据 → 报错

## 五、AI 执行清单

分析一份报告样本时，按以下顺序执行：

### 阶段 1：结构分析
- [ ] 通读全文，列出所有段落和表格
- [ ] 区分固定文本和可变数据
- [ ] 识别不固定行数表格 → `{%tr for %}`
- [ ] 识别条件内容 → `{%p if %}`

### 阶段 2：命名设计
- [ ] 为键值对数据分组建 Sheet
- [ ] 为循环表格命名 Sheet
- [ ] 字段编码使用中文
- [ ] 布尔变量以"有"开头

### 阶段 3：模板制作
- [ ] 可变值 → `{{ Sheet.字段 }}`
- [ ] 添加过滤器（money / percent / num / date）
- [ ] 不固定表格 → `{%tr for %}...{%tr endfor %}`
- [ ] 条件块 → `{%p if %}...{%p endif %}`
- [ ] 合计行放在 `{%tr endfor %}` 之后

### 阶段 4：数据字典制作
- [ ] 每个键值对 Sheet：第一行 `字段编码 │ 值`
- [ ] 每个表格 Sheet：第一行为列标题
- [ ] 布尔值默认 `FALSE`
- [ ] 数值列设置单元格格式为数值

### 阶段 5：验证
- [ ] 每个 `{{ }}` 在 Excel 中都有对应字段
- [ ] Sheet 名与模板引用一致
- [ ] `{%p if %}` / `{%p endif %}` 成对
- [ ] `{%tr for %}` / `{%tr endfor %}` 成对

## 六、常见问题

**Q: 合计行放在哪里？**
A: 放在 `{%tr endfor %}` 之后。如需动态合计，将合计值作为独立键值对变量。

**Q: 什么算"不固定"？什么算"有条件"？**
A: "不固定"指数据行数会变化（如股东 1-10 人）。"有条件"指某些段落仅在部分报告中存在。

**Q: 日期格式如何统一？**
A: Excel 中统一用 `YYYY-MM-DD`，模板中可选 `| date` 转中文格式。

**Q: 金额为 0 时显示什么？**
A: 金额为 0 时 `| money` 输出空。如需显示 "0.00"，直接用 `{{ 变量 }}` 不加过滤器。
