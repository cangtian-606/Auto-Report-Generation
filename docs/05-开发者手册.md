## 一、系统架构

```
┌──────────────────────────────────────────────────────────────────┐
│  src/orchestrator.py                                             │
│  编排层：RenderPipeline 四个阶段式执行，临时文件统一管理          │
└──────┬──────┬──────┬─────────────────────────────────────────────┘
       │      │      │
       ▼      ▼      ▼
┌──────────┐ ┌──────────────┐ ┌──────────────┐
│ reader/  │ │ processing/  │ │ render/      │
│ xlsx.py  │ │ mapper.py    │ │ generator.py │
│ yaml.py  │ │ schema.py    │ │ filters.py   │
└──────────┘ │ table_       │ │ pipeline.py  │ ← RenderPipeline
              │ preprocessors│ │ context.py   │ ← RenderContext
              │ docx_editor  │ ← DocxEditor │ analyzer.py  │ ← TemplateAnalyzer
              └──────────────┘ └──────────────┘

src/
├── __main__.py         # python -m src 入口
├── orchestrator.py     # 编排：四阶段流水线 + 摘要输出
├── cli.py              # CLI 入口
├── path_guard.py       # 路径安全校验
├── logging_config.py   # 日志配置（run_id + 阶段耗时追踪）
├── exceptions.py       # 自定义异常
├── reader/             # 数据读取层
├── processing/         # 数据处理与校验层
│   ├── mapper.py       # DataMapper：原始数据 → context
│   ├── schema.py       # SchemaValidator：context 完整性校验
│   ├── table_preprocessors.py  # Tc/Tr 继承预处理器
│   └── docx_editor.py  # DocxEditor：zip 解/打包上下文管理器
└── render/             # 模板渲染层
    ├── generator.py    # DocumentGenerator：docxtpl 封装
    ├── filters.py      # 自定义过滤器
    ├── pipeline.py     # RenderPipeline：阶段编排 + 临时文件管理
    ├── context.py      # RenderContext：构建 + 扁平化 + 校验
    └── analyzer.py     # TemplateAnalyzer：模板自省（stats/变量）
```

### 数据流

```
Excel/YAML → Reader → RenderContext.build()
                              │
                    ┌─────────┘
                    ▼
            DataMapper + 扁平化
                    │
                    ▼
            ┌──────────────────┐
            │ RenderPipeline    │
            │ ┌────────────────┐ │
            │ │ 1. 读数据       │ │
            │ │ 2. 构建上下文   │ │
            │ │ 3. 预处理器     │ │
            │ │    Tr → Tc      │ │
            │ │ 4. 渲染         │ │
            │ │    TemplateAnalyzer │
            │ │    DocumentGenerator │
            │ └────────────────┘ │
            └──────────────────┘
                    │
                    ▼
              output.docx
```

### 分层职责

| 层 | 模块 | 职责 |
|----|------|------|
| 数据读取 | `reader/xlsx.py`, `reader/yaml.py` | 读取 Excel/YAML → 原始数据字典 |
| 数据处理 | `processing/mapper.py` | 原始数据 → Jinja2 context（嵌套、类型转换） |
| 数据校验 | `processing/schema.py` | Schema 定义 + 对 context 进行完整性校验 |
| 上下文构建 | `render/context.py` | RenderContext：构建 + 扁平化 + 校验聚合 |
| 模板预处理 | `processing/table_preprocessors.py` | TrInheritance：自动补全 `{%tr endfor %}`；TcInheritance：首行 `{%tc for %}` 声明自动注入后续行 |
| docx I/O | `processing/docx_editor.py` | DocxEditor：zip 解包/打包上下文管理器，预处理器不再管理临时文件 |
| 模板分析 | `render/analyzer.py` | TemplateAnalyzer：模板 stats（表格数/标签数）+ 变量检查（未定义/未使用） |
| 模板渲染 | `render/generator.py`, `render/filters.py` | SandboxedEnvironment 渲染, 自定义过滤器 |
| 流水线 | `render/pipeline.py` | RenderPipeline：阶段序列执行 + 临时文件生命周期管理 |
| 编排 | `orchestrator.py` | 组装各模块为四阶段流水线，提供 `generate()` 一站式入口 |
| CLI | `cli.py` | 参数解析、路径校验、调度 |

## 二、模块说明

### 2.1 Reader 层

**ExcelDataReader**（[reader/xlsx.py](file:///d:/ProgramWorkingSpace/Auto-Report-Generation/src/reader/xlsx.py)）

- 使用 `pandas` + `openpyxl` 读取 `.xlsx`
- 自动识别 Sheet 类型：首行为 `字段编码` → 键值对，否则 → 表格
- `read_all()` 返回 `Dict[str, Union[dict, DataFrame]]`

**YamlDataReader**（[reader/yaml.py](file:///d:/ProgramWorkingSpace/Auto-Report-Generation/src/reader/yaml.py)）

- 读取 `.yaml` → 直接返回嵌套结构

**工厂函数**（[reader/__init__.py](file:///d:/ProgramWorkingSpace/Auto-Report-Generation/src/reader/__init__.py)）

```python
from src.reader import create_reader
reader = create_reader("data.xlsx")  # 自动选择 ExcelDataReader 或 YamlDataReader
```

### 2.2 Processing 层

**DataMapper**（[processing/mapper.py](file:///d:/ProgramWorkingSpace/Auto-Report-Generation/src/processing/mapper.py)）

核心映射逻辑：
- `build_context()` 遍历所有 Sheet，按类型分派
- 键值对 Sheet → `context[sheet_name] = {key: value}`
- 表格 Sheet → `context[sheet_name] = [{col: val}]`
- 嵌套 Sheet（含 `.`）→ 解析父子关系，自动挂载子数据
- `_convert_value()` 统一处理 None→""、"TRUE"/"FALSE"→bool

**SchemaValidator**（[processing/schema.py](file:///d:/ProgramWorkingSpace/Auto-Report-Generation/src/processing/schema.py)）

```python
validator = SchemaValidator()
validator.load_from_file("schema.json")
errors = validator.validate(context)
```

**TablePreprocessors**（[processing/table_preprocessors.py](file:///d:/ProgramWorkingSpace/Auto-Report-Generation/src/processing/table_preprocessors.py)）

渲染前自动处理模板 XML，通过 `DocxEditor` 管理 zip 生命周期：

| 预处理器 | 触发条件 | 行为 |
|---------|---------|------|
| `TcInheritancePreprocessor` | 表格首行含 `{%tc for %}` | 后续行自动注入列循环标签，"首行声明，整表继承" |
| `TrInheritancePreprocessor` | 表格中含 `{%tr for %}` 但缺 `{%tr endfor %}` | 在表末插入新行写入 `{%tr endfor %}` |

执行顺序：Tr 先于 Tc，通过 `orchestrator._preprocess_template()` 串联。

**DocxEditor**（[processing/docx_editor.py](file:///d:/ProgramWorkingSpace/Auto-Report-Generation/src/processing/docx_editor.py)）

```python
from src.processing.docx_editor import DocxEditor

with DocxEditor("template.docx") as editor:
    root = editor.root          # lxml 解析后的 XML 根元素
    # 修改 XML 树...
    editor.save_to("output.docx", modified=True)
```

将 zip 解包、XML 解析、写回、重新打包封装在 with 块中，预处理器不再管理临时目录和 zip 操作。

### 2.3 Render 层

**DocumentGenerator**（[render/generator.py](file:///d:/ProgramWorkingSpace/Auto-Report-Generation/src/render/generator.py)）

```python
gen = DocumentGenerator("template.docx")
gen.render(context, "output.docx")
```

- 使用 `TemplateAnalyzer` 获取共享的 `SandboxedEnvironment`（消除重复创建）
- 变量检查和语法分析已移至 `TemplateAnalyzer`
- `_template_cache` 缓存已加载模板

**TemplateAnalyzer**（[render/analyzer.py](file:///d:/ProgramWorkingSpace/Auto-Report-Generation/src/render/analyzer.py)）

```python
from src.render.analyzer import TemplateAnalyzer

analyzer = TemplateAnalyzer("template.docx")
tables, tags = analyzer.get_stats()          # 模板结构统计
undeclared = analyzer.get_undeclared(ctx)     # 未定义变量检测
unused = analyzer.get_unused(ctx)            # 未使用数据字段检测
jinja_env = analyzer.jinja_env               # 共享的 SandboxedEnvironment
```

聚合模板自省能力，加载模板一次即可执行多种分析。

**RenderContext**（[render/context.py](file:///d:/ProgramWorkingSpace/Auto-Report-Generation/src/render/context.py)）

```python
from src.render.context import RenderContext

rctx = RenderContext(raw_data)
context = rctx.build()              # 构建 + 扁平化
errors = rctx.validate(schema_path) # 可选校验
```

将 context 构建（DataMapper）、子表扁平化、Schema 校验聚合为一个深度模块。

**RenderPipeline**（[render/pipeline.py](file:///d:/ProgramWorkingSpace/Auto-Report-Generation/src/render/pipeline.py)）

```python
from src.render.pipeline import RenderPipeline

pipeline = RenderPipeline()
pipeline.add_stage("读数据", read_fn)
pipeline.add_stage("构建", build_fn)
state = pipeline.run()
pipeline.cleanup_temps()  # 统一清理临时文件
```

管理阶段顺序执行、临时文件生命周期，为将来的并行批量渲染提供基础。

**自定义过滤器**（[render/filters.py](file:///d:/ProgramWorkingSpace/Auto-Report-Generation/src/render/filters.py)）

7 个过滤器：`money`, `percent`, `num`, `date`, `int`（别名 `number`）, `default_dash`, `default`

> 完整过滤器表见 [02-数据格式规范](./02-数据格式规范.md#七过滤器速查)

### 2.4 编排层

**orchestrator.py**（[orchestrator.py](file:///d:/ProgramWorkingSpace/Auto-Report-Generation/src/orchestrator.py)）

```python
from src.orchestrator import generate
generate("data.xlsx", "template.docx", "output.docx", strict=False)
```

`generate()` 组装四阶段流水线：

| 阶段 | 操作 | 耗时记录 |
|------|------|---------|
| 读数据 | `create_reader()` + `read_all()` | ✓ |
| 构建 | `RenderContext.build()` (Mapper + 扁平化) | ✓ |
| 预处理 | `TrInheritance` → `TcInheritance` + `TemplateAnalyzer.get_stats()` | ✓ |
| 渲染 | `TemplateAnalyzer.get_undeclared()` → `DocumentGenerator.render()` | ✓ |

每次运行输出汇总摘要：

```
──────────────────────────────────────────────────────
  运行 ID: 778b91bd
  数据文件: 数据.xlsx · 23 Sheet · 63 行
  模板文件: 模板.docx · 14 表格 · 156 标签
  输出文件: output.docx
  执行结果: ✓ 成功
  阶段耗时: 1.4s = 读数据0.2 + 构建0.0 + 预处理0.3 + 渲染0.9
  变量检查: 0 未定义
```

### 2.5 异常体系

| 异常 | 层级 | 用途 |
|------|------|------|
| `DataReadError` | Reader / Processing | 数据读取/解析/映射失败 |
| `TemplateError` | 基类 | 模板错误的基类 |
| `TemplateSyntaxError` | Render | 模板语法/变量缺失 |
| `ValidationError` | Processing | Schema 校验失败 |

CLI 层统一捕获 `(TemplateError, ValidationError, DataReadError)`。

## 三、核心 API

```python
from src.orchestrator import generate
from src.render.generator import DocumentGenerator
from src.render.analyzer import TemplateAnalyzer
from src.render.context import RenderContext
from src.render.pipeline import RenderPipeline
from src.reader import create_reader

# 一站式
generate("data.xlsx", "template.docx", "output.docx")

# 分步使用深度模块
reader = create_reader("data.yaml")
rctx = RenderContext(reader.read_all())
context = rctx.build()
rctx.validate("schema.json", strict=True)

analyzer = TemplateAnalyzer("template.docx")
undeclared = analyzer.get_undeclared(context)

gen = DocumentGenerator("template.docx")
gen.render(context, "output.docx")
```

### 嵌套数据示例

```python
# DataMapper 通过 Sheet 命名中的 "." 表达嵌套
# Sheet "EPC合同" → 父列表, Sheet "EPC合同.epc金额" → 子表格（通过 _parent_ 关联）
raw = {
    "EPC合同": pd.DataFrame({"项目名称": ["深圳光伏", "广州光伏"], "合同金额": [500, 300]}),
    "EPC合同.epc金额": pd.DataFrame({
        "_parent_项目名称": ["深圳光伏", "深圳光伏", "广州光伏"],
        "合同分项": ["组件", "逆变器", "组件"],
        "分项金额": [300, 100, 200],
    }),
}
rctx = RenderContext(raw)
ctx = rctx.build()
# ctx["EPC合同"][0]["epc金额"] → [{"合同分项": "组件", "分项金额": 300}, {"合同分项": "逆变器", "分项金额": 100}]
# 扁平化后: ctx["EPC合同.epc金额"] → [...] （所有子项合并的顶级列表）

# 键值对型父表下的子表（无 _parent_ 列）
raw2 = {
    "历史沿革": {"注册资本": 1000},
    "历史沿革.股权结构": pd.DataFrame({"股东名称": ["张伟", "李芳"]}),
}
ctx2 = RenderContext(raw2).build()
# ctx2["历史沿革"]["股权结构"] → [{"股东名称": "张伟"}, {"股东名称": "李芳"}]
```

## 四、扩展开发

### 添加新过滤器

```python
# src/render/filters.py
def filter_rmb(value):
    if value is None or value == '':
        return ''
    return f"¥{float(value):,.2f}"

FILTERS['rmb'] = filter_rmb
```

### 添加新数据源

创建新 Reader 类，实现 `read_all() → Dict`，在 `reader/__init__.py` 的 `create_reader` 中注册。

### 添加新预处理步骤

在 `orchestrator._preprocess_template()` 中追加新预处理器，预处理器只需实现 `process(input_path, output_path) → int`，内部使用 `DocxEditor` 管理 docx I/O。

### 添加 Schema 验证

```json
{
  "entries": {
    "全局信息": {
      "required": true,
      "fields": {
        "公司名": {"required": true, "field_type": "string"}
      }
    }
  }
}
```

## 五、日志与调试

系统通过 `src/logging_config.py` 集中配置日志。

### 默认行为

运行时自动生成日志文件 `logs/generate_YYYYMMDD_HHMMSS.log`，完整记录 DEBUG 及以上日志。

### CLI 控制

| 参数 | 效果 |
|------|------|
| （默认） | 终端 INFO，文件 DEBUG，自动保存到 `logs/` |
| `--verbose` / `-v` | 终端显示 DEBUG 输出 |
| `--quiet` / `-q` | 终端仅显示 WARNING+ |
| `--log-file PATH` | 指定日志文件路径 |
| `--no-log` | 禁用文件日志，仅终端输出 |

### 结构化日志

每条日志包含运行 ID 和时间戳：

```
2026-05-27 10:30:15 [778b91bd] [   INFO] 开始生成: 数据.xlsx → output.docx
2026-05-27 10:30:16 [778b91bd] [   INFO]  执行结果: ✓ 成功
2026-05-27 10:30:16 [778b91bd] [   INFO]  阶段耗时: 1.4s = 读数据0.2 + 构建0.0 + 预处理0.3 + 渲染0.9
```

### 编程接口

```python
from src.logging_config import init_run_id, get_run_id, timed_stage, get_stage_times

init_run_id()
with timed_stage("我的阶段"):
    do_work()
times = get_stage_times()  # {"我的阶段": 1.23}
```

## 八、部署

### 依赖

```bash
pip install -r requirements.txt
```

依赖清单：`docxtpl`, `jinja2`, `pandas`, `openpyxl`, `python-docx`, `lxml`, `pyyaml`

### 目录约定

```
data/          ← 数据文件（.xlsx / .yaml）
templates/     ← Word 模板（.docx）
output/        ← 输出目录
logs/          ← 运行日志（自动生成）
```

`validate_path()` 将用户输入路径限制在上述目录及项目根目录内。
