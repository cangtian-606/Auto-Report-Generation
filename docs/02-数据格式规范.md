# 数据格式规范

> 本文档是数据格式的**唯一事实源**。其他文档通过引用本文档来避免重复。

---

## 一、数据文件整体结构

系统支持 **Excel**（`.xlsx` / `.xls`）和 **YAML**（`.yaml` / `.yml`）两种格式，根据扩展名自动识别。

### Excel 结构

```
Excel 文件
├── Sheet: 全局              ← 键值对（封面字段）
├── Sheet: 释义              ← 行循环表格
├── Sheet: 历史沿革           ← 键值对（公司信息）
├── Sheet: 历史沿革.股权结构   ← 子表格（嵌套在历史沿革下）
├── Sheet: 项目基本情况         ← 列表（段落循环用）
├── Sheet: EPC合同            ← 列表 + 子表（嵌套循环用）
├── Sheet: EPC合同.epc金额    ← 列表内子表格
└── Sheet: 税率政策           ← 键值对 + 布尔控制
```

### YAML 结构

```yaml
全局:
  公司名称: 深圳市创新科技有限公司
  报告文号: "2025-001"
释义:
  - 全称: 大华会计师事务所
    简称: 大华
历史沿革:
  注册资本: 1000
  成立日期: "2018-06-15"
  股权结构:
    - 股东名称: 张伟
      认缴出资额: 600
项目基本情况:
  - 项目名称: 深圳分布式光伏项目
    项目地址: 深圳市宝安区
    项目装机量: 5MW
税率政策:
  有三免三减半: true
```

## 二、四种模板标签类型

| 类型 | Excel 表现 | 模板引用方式 | 用途 |
|------|-----------|-------------|------|
| 键值对 | 两列：字段编码、值 | `{{ Sheet名.字段名 }}` | 单个值（名称、日期、金额等） |
| 行循环（表格） | 列标题 + 数据行 | `{%tr for i in Sheet名 %}` | 行数不固定的表格行 |
| 段落循环 | 列表数据 | `{% for i in Sheet名 %}` | 行数不固定的段落块 |
| 列循环（表格） | 嵌套对象列表 | `{%tc for i in Sheet名 %}` | 列数不固定的表格列 |

## 三、键值对格式

### Excel

| 字段编码 | 值 |
|----------|-----|
| 公司名称 | 深圳市创新科技有限公司 |
| 注册资本 | 1000 |
| 成立日期 | 2018-06-15 |
| 有三免三减半 | TRUE |

**规则**：
- 第一行是标题行，系统自动跳过
- A 列写字段编码（只写字段名，Sheet 名自动作为前缀）
- B 列写值
- 布尔值用 `TRUE` / `FALSE`（不区分大小写）
- 数值直接填数字，文本直接填文字

### YAML

```yaml
全局:
  公司名: 深圳市创新科技有限公司
  注册资本: 1000
  成立日期: "2018-06-15"
  有职工信息: true
```

## 四、表格（循环列表）格式

### 4.1 行循环表格 `{%tr for %}`

用在表格行数不固定的场景，将一行数据行作为模板，循环渲染 N 行。

#### Excel

| 股东 | 认缴金额 | 认缴比例 |
|------|---------|---------|
| 张伟 | 600     | 0.60    |
| 李芳 | 250     | 0.25    |
| 王强 | 150     | 0.15    |

**规则**：
- 第一行是列标题
- 每行一条数据
- 循环变量统一使用 `i`
- 模板语法：`{%tr for i in Sheet名 %}` ... `{%tr endfor %}`

#### 模板写法

```
┌──────┬──────────────────┬────────────────────┐
│ 股东 │ 认缴金额          │ 认缴比例             │
├──────┼──────────────────┼────────────────────┤
│ {%tr for i in 历史沿革.股权结构 %}                │
├──────┼──────────────────┼────────────────────┤
│ {{ i.股东 }} │ {{ i.认缴金额\|num }} │ {{ i.认缴比例\|percent }} │
├──────┼──────────────────┼────────────────────┤
│ {%tr endfor %}                                   │
└──────┴──────────────────┴────────────────────┘
```

### 4.2 嵌套行循环（列表内子表）

当列表中的每个项包含子列表时使用：

```
{% for i in EPC合同 %}
  EPC承包单位：{{ i.承包单位 }}
  合同金额明细：
  ┌────────┬──────────┬─────────┬──────────┐
  │ 序号   │ 合同分项  │ 税率    │ 金额     │
  ├────────┼──────────┼─────────┼──────────┤
  │ {%tr for k in i.epc金额 %}                   │
  ├────────┼──────────┼─────────┼──────────┤
  │ {{ k.序号\|int }} │ {{ k.合同分项 }} │ {{ k.税率\|percent }} │ {{ k.分项金额\|num }} │
  ├────────┼──────────┼─────────┼──────────┤
  │ {%tr endfor %}                               │
  ├────────┼──────────┼─────────┼──────────┤
  │ 合计   │          │         │ {{ i.合同金额\|num }} │
  └────────┴──────────┴─────────┴──────────┘
{% endfor %}
```

### 4.3 段落循环 `{% for %}`

当需要循环的不是表格行而是整个段落块时使用：

```
{% for i in 项目基本情况 %}

{{ i.项目名称 }}
  地址：{{ i.项目地址 }}
  装机容量：{{ i.项目装机量 }}
  模式：{{ i.项目模式 }}

{% endfor %}
```

**规则**：`{% for %}` 和 `{% endfor %}` 各占独立一个段落，之间内容被整体复制 N 次。

### 4.4 列循环表格 `{%tc for %}`

当表格列数不固定时使用，详见 [04-模板制作规范](./04-模板制作规范.md#41-列循环-tc-for)。

## 五、命名规范

### Sheet 命名

使用中文语义名称，见名知义。以下为常见命名风格：

| Sheet 名 | 类型 | 用途 |
|----------|------|------|
| 全局 | 键值对 | 封面/全局字段 |
| 释义 | 表格 | 全称-简称对照表 |
| 历史沿革 | 键值对 + 子表 | 公司基本信息及股权结构 |
| 项目基本情况 | 列表 | 多个项目的核心参数 |
| EPC合同 | 列表 + 子表 | EPC工程合同及金额明细 |
| EMC合同 | 列表 | EMC能源管理合同 |
| 购售电合同 | 列表 | 购售电合同信息 |
| 税率政策 | 键值对 | 税率及优惠政策 |

### 字段命名原则

1. 使用中文，见名知义
2. 同一报告内变量名保持唯一
3. 相似概念归入同一 Sheet
4. 布尔控制字段以"有"或"是否"开头（如 `有三免三减半`、`有小微企业`）

### 变量引用语法

```
Sheet: 全局
  字段编码: 公司简称   → 模板变量: {{ 全局.公司简称 }}

Sheet: 历史沿革（键值对）
  字段编码: 注册资本   → 模板变量: {{ 历史沿革.注册资本 }}

Sheet: 历史沿革.股权结构（子表格）
  列标题: 股东名称     → 嵌套循环变量: {{ i.股东名称 }}
  循环写法: {%tr for i in 历史沿革.股权结构 %}

Sheet: 项目基本情况（段落循环）
  列表项属性: 项目地址 → 循环变量: {{ i.项目地址 }}
  循环写法: {% for i in 项目基本情况 %}

Sheet: EPC合同.epc金额（列表内子表）
  列标题: 合同分项     → 嵌套循环变量: {{ k.合同分项 }}
  循环写法: {%tr for k in i.epc金额 %}
```

## 六、嵌套数据

通过 Sheet 命名中的 `.` 表达父子关系。系统在构建上下文时会自动处理两种嵌套算法：

### 6.1 表格型父表（_parent_ 关联）

**父 Sheet**：`项目公司`（列表型父数据）

| 公司简称 | 注册资本 |
|---------|---------|
| 重庆晟和泰 | 1000 |
| 安徽富军 | 800 |

**子 Sheet**：`项目公司.股东出资`（通过 `_parent_` 列关联）

| _parent_公司简称 | 股东 | 认缴金额 |
|-----------------|------|---------|
| 重庆晟和泰 | 张伟 | 600 |
| 重庆晟和泰 | 李芳 | 250 |
| 安徽富军 | 王强 | 800 |

**规则**：
- `父Sheet.子Sheet`：点号分隔
- 子表首列必须以 `_parent_` 开头，值匹配父表的同名列
- 子表自动挂载为父列表中每项的属性：`item["股东出资"] = [...]`
- 子行找不到父行 → 报错，父行无子数据 → 报错

### 6.2 键值对型父表（字段挂载）

**父 Sheet**：`历史沿革`（键值对型）

**子 Sheet**：`历史沿革.股权结构`（无 `_parent_` 列的表格）

| 股东名称 | 认缴出资额 |
|---------|----------|
| 张伟 | 600 |
| 李芳 | 250 |

子表自动挂载为：`context["历史沿革"]["股权结构"] = [...]`

### 6.3 自动扁平化

对于列表型父表下的子表，系统还会**自动扁平化一份顶级访问路径**，方便模板直接用简写路径访问：

```python
# 原始: context["EPC合同"] = [{"epc金额": [...]}, ...]
# 扁平化后同时存在: context["EPC合同.epc金额"] = [...]
```

这样模板中两种写法都有效：

```
{%tr for k in i.epc金额 %}          ← 嵌套在父循环内用属性路径
{%tr for k in EPC合同.epc金额 %}    ← 顶级扁平化路径，独立使用
```

### YAML

```yaml
项目公司:
  - 公司简称: 重庆晟和泰
    股东出资:
      - 股东: 张伟
        认缴金额: 600
```

YAML 天然支持嵌套。

## 七、过滤器速查

> 这是过滤器的**唯一定义源**。修改过滤器逻辑请更新 `src/render/filters.py`。

| 过滤器 | 输入示例 | 输出 | 说明 |
|--------|---------|------|------|
| `\| num` | 1234567.89 | 1,234,567.89 | 千分位金额，默认 2 位小数，0 / 空 → "" |
| `\| num(0)` | 1234567.89 | 1,234,568 | 0 位小数 |
| `\| num(2, '万')` | 12345678 | 1,234.57 | 除以万，2 位小数 |
| `\| percent` | 0.25 | 25.00% | 小数自动乘 100，0 → "" |
| `\| date` | 2025-01-15 | 2025年01月15日 | 支持 `-`、`/`、年月日 格式 |
| `\| int` | 123.45 | 123 | 截断取整 |
| `\| default_dash` | None/"" | "" | 空值显示为空，非空返回字符串 |
| `\| default('-')` | None/"" | - | 空值用自定义字符串替换 |
| `\| paragraphs` | 第一行\n第二行 | Word 段落换行 | 将 \n 替换为真实段落，保留首行缩进 |

**`| num` 参数**：`decimals` 为小数位数（默认 2），负数表示整数位精度（-2=精确到百位）；`unit` 为单位（十/百/千/万/十万/百万/千万/亿/十亿）。

**`| money` 保留为 `| num` 别名**，现有模板无需改动。

**重要**：`num`、`percent` 对 0 值返回空字符串。如需显示 "0"，直接输出数值不加过滤器。

## 八、布尔控制字段

用于控制条件显示：

| Excel 值 | 模板用法 | 效果 |
|---------|---------|------|
| TRUE | `{%p if 税率政策.有三免三减半 %}` | 显示该块 |
| FALSE | `{%p if 税率政策.有三免三减半 %}` | 隐藏该块 |
| 不填 | `{%p if 税率政策.有三免三减半 %}` | 隐藏该块 |

TRUE / FALSE 不区分大小写，前后空格自动去除。
