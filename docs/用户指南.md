## 二、Excel数据填写规范

### 2.1 基本格式

Excel文件包含一个或多个Sheet，每个Sheet对应模板中的不同数据域。

**Sheet命名规则**：

| Sheet名称 | 用途 | 数据格式 |
|-----------|------|---------|
| `date.全局信息` | 键值对（公司名、日期等） | 键值对 |
| `date.基本情况` | 键值对（条件标志、数值等） | 键值对 |
| `form.股东出资` | 循环表格 | 表格 |
| `form.投资明细` | 循环表格 | 表格 |

### 2.2 键值对Sheet格式

适用于单个变量的值：

| 字段编码 | 值 |
|----------|-----|
| 公司名 | 公司名称 |
| 日期 | 2025-01-15 |
| 备注 | 这是备注 |
| 有职工信息 | TRUE |

**规则**：
- 第一行是标题（自动跳过）
- A列：字段编码（只写字段名，系统自动与Sheet名前缀组合）
- B列：值
- 布尔值用 `TRUE`/`FALSE`（大写）
- 数值直接填写数字
- 文本直接填写文字

### 2.3 表格Sheet格式

适用于循环生成的数据：

**Sheet名称**：`form.股东出资`（对应模板中的 `{{ form.股东出资 }}`）

| 股东 | 认缴金额 | 认缴比例 |
|------|---------|---------|
| 李芳 | 250 | 0.25 |
| 王强 | 250 | 0.25 |

**规则**：
- 第一行是列标题（对应变量属性名）
- 后续行是数据
- 数值列直接填数字
- 空值留空

## 三、变量命名参考

### 3.1 两种数据类型

| 类型 | Sheet前缀 | 模板引用 | 用途 |
|------|-----------|---------|------|
| 键值对 | `date.xxx` | `{{ date.字段 }}` | 单个变量值 |
| 表格 | `form.xxx` | `{%tr for item in form.名称 %}` | 循环数据列表 |

### 3.2 过滤器使用

在变量后加 `| 过滤器` 进行格式化：

| 过滤器 | 效果 | 示例输入 | 输出 |
|--------|------|---------|------|
| `money` | 千分位金额 | `1234567.89` | `1,234,567.89` |
| `percent` | 百分比（自动乘100） | `0.25` | `25.00%` |
| `num` | 千分位数字 | `1234567` | `1,234,567` |
| `date` | 日期格式化 | `2025-01-15` | `2025年01月15日` |
| `default('-')` | 空值默认 | `''` | `-` |

**使用示例**：
```
注册资本：{{ date.基本情况.注册资本 | num }}万元
比例：{{ i.认缴比例 | percent }}
备注：{{ date.备注 | default('无') }}
```

## 四、命令执行

### 4.1 单文件生成

```bash
python -m src.generator --data 数据.xlsx --template 模板.docx --output 输出.docx
```

**简化写法**：
```bash
python -m src.generator -d 数据.xlsx -t 模板.docx -o 输出.docx
```

### 4.2 批量生成

```bash
python -m src.generator --batch 数据目录/ --template 模板.docx --output-dir 输出目录/
```

**简化写法**：
```bash
python -m src.generator -b 数据目录/ -t 模板.docx --output-dir 输出目录/
```

### 4.3 常用选项

| 选项 | 说明 |
|------|------|
| `-d, --data FILE` | Excel数据文件 |
| `-t, --template FILE` | Word模板文件 |
| `-o, --output FILE` | 输出文件路径（单文件模式） |
| `-b, --batch DIR` | 批量模式：数据文件目录 |
| `--output-dir DIR` | 批量模式：输出目录 |
| `--strict` | 严格模式：未定义变量时报错退出 |
| `--no-check` | 跳过变量检查 |
| `--schema FILE` | Schema 验证文件路径 (.json) |
| `--validate` | 启用数据验证 |
| `--strict-validate` | 严格验证：验证失败时报错退出 |
| `--check-syntax` | 渲染前检查模板语法 |
| `--report-unused` | 报告数据中未被模板使用的字段 |

## 五、输出检查清单

生成文档后，检查以下内容：

- [ ] 所有变量是否已替换（非空或符合预期）
- [ ] 条件显示是否正确（显示/隐藏符合预期）
- [ ] 循环数据行数是否正确
- [ ] 金额/百分比格式是否正确
- [ ] 页眉页脚是否正确
- [ ] 页面布局是否保持

## 六、错误处理

### 常见错误

| 错误信息 | 原因 | 解决方法 |
|---------|------|---------|
| `未定义变量: xxx` | 数据中缺少该字段 | 在Excel中添加该字段 |
| `找不到Sheet: xxx` | Sheet名称不匹配 | 检查Sheet命名 |
| `渲染失败` | 模板语法错误 | 检查模板变量格式 |

### 数据校验

使用 `--no-check` 可跳过变量检查，强制生成：

```bash
python -m src.generator -d data.xlsx -t template.docx -o output.docx --no-check
```

业务人员通常不需要制作模板，模板由IT部门或模板管理员提供。

**获取模板后**：
1. 确认模板版本
2. 了解所需填写的数据字段
3. 按照本手册填写Excel数据
4. 执行生成命令

**如需新模板或修改现有模板**，请联系IT部门。

## 九、键值对Sheet详解

### 9.1 格式说明

适用于单个变量的值（如公司名称、日期、金额等）。

**Sheet名称**：`date.全局信息`

```
| 字段编码 | 值 |
|----------|-----|
| 公司名 | 北京科技有限公司 |
| 日期 | 2025-01-15 |
| 注册资本 | 1000 |
```

**规则**：
- 第一行自动识别为标题并跳过
- A列：字段编码（只写字段名，系统自动组合为 `date.全局信息.公司名`）
- B列：值
- 支持中文列名

### 9.2 数据类型

#### 字符串
```
| 字段编码 | 值 |
|----------|-----|
| date.名称 | 公司名称 |
```
直接填写文字。

#### 数字
```
| 字段编码 | 值 |
|----------|-----|
| date.年份 | 2025 |
| date.金额 | 10000.50 |
```
填写数字，会自动识别为数值类型。

#### 布尔值
```
| 字段编码 | 值 |
|----------|-----|
| date.显示 | TRUE |
| date.有附件 | FALSE |
```
使用 `TRUE`/`FALSE`（大写）。

#### 日期
```
| 字段编码 | 值 |
|----------|-----|
| date.日期 | 2025-01-15 |
| date.签订日期 | 2025年1月15日 |
```
支持多种日期格式。

## 十、表格型Sheet详解

### 10.1 格式说明

适用于循环数据（如商品列表、人员列表等）。

**Sheet命名**：`form.名称`（对应模板中的 `{{ form.名称 }}`）

```
| 名称 | 数量 | 单价 | 金额 |
|------|------|------|------|
| 产品A | 100 | 50.00 | 5000.00 |
| 产品B | 50 | 80.00 | 4000.00 |
```

**规则**：
- 第一行是列标题
- 列标题对应循环项的属性名
- 后续行是数据

### 10.2 命名对应关系

| Sheet命名 | 模板引用 |
|-----------|---------|
| `form.股东出资` | `{%tr for i in form.股东出资 %}` |
| `form.投资明细` | `{%tr for i in form.投资明细 %}` |
| `form.项目明细` | `{%tr for i in form.项目明细 %}` |

循环变量统一使用 `i`。

### 10.3 多Sheet对应

一个Excel可以有多个表格型Sheet：

```
Sheet: form.产品列表  →  {{ form.产品列表 }}
Sheet: form.服务列表  →  {{ form.服务列表 }}
```

## 十一、数据映射规则

### 11.1 自动映射

系统根据Sheet名和字段编码自动分配，无需手动指定前缀：

```
Sheet: date.全局信息
字段编码: 公司名
        ↓
模板变量: {{ date.全局信息.公司名 }}

Sheet: date.基本情况
字段编码: 注册资本
        ↓
模板变量: {{ date.基本情况.注册资本 }}
```

字段编码只写字段名（如 `公司名`），`date.` 前缀由Sheet名自动提供。

### 11.2 表格数据映射

表格Sheet直接映射为数据列表：

```
Sheet: form.商品列表 (名称, 数量, 单价, 金额)
        ↓
context['form']['商品列表'] = [
    {名称: "产品A", 数量: 100, 单价: 50.00, 金额: 5000.00},
    {名称: "产品B", 数量: 50, 单价: 80.00, 金额: 4000.00}
]
```

### 11.3 布尔值转换

| Excel填写 | Python值 | 条件渲染结果 |
|-----------|---------|------------|
| `TRUE` | `True` | 显示 |
| `FALSE` | `False` | 隐藏 |
| `true` / `TRUE` | `True` | 显示 |
| 空字符串 `''` | `''` | 隐藏（空值） |

系统会自动将 Excel 中的 `"TRUE"`/`"FALSE"` 字符串转换为 Python `True`/`False`，无需手动处理。

### 12.1 必填字段

## 十六、嵌套数据填写（高级）

当一个公司下面有多张子表（如股东出资、资产清单、贷款明细），使用嵌套 Sheet 结构。

### 父 Sheet（列表）

| 公司简称 | 公司名称 | 信用代码 |
|----------|---------|---------|
| 重庆晟和泰 | 重庆晟和泰新能源科技有限公司 | 91500227MA5U7XXX |
| 安徽富军 | 安徽富军新能源有限公司 | 91341525MA2TXXXX |

### 子 Sheet（点号命名，首列用 `_parent_` 关联）

Sheet名：`form.项目公司.股东出资`

| _parent_公司简称 | 股东 | 认缴额 | 比例 |
|-----------------|------|--------|------|
| 重庆晟和泰 | 张伟 | 600 | 0.60 |
| 重庆晟和泰 | 李芳 | 250 | 0.25 |
| 安徽富军 | 王强 | 500 | 0.50 |

Sheet名：`form.项目公司.资产清单`

| _parent_公司简称 | 资产名称 | 原值 |
|-----------------|---------|------|
| 重庆晟和泰 | 光伏设备 | 2681250 |
| 安徽富军 | 光伏设备 | 1875000 |

### 关键规则

| 规则 | 说明 |
|------|------|
| 子 Sheet 名 | `父Sheet名.子名`，点号分隔 |
| `_parent_` 首列 | 必须为 `_parent_父列名`，值精确匹配父表某列 |
| 不能留空 | 父表每行都必须有子数据，子行必须能找到父行，否则报错 |
| 多层嵌套 | 支持 `form.父.子.孙`，继续用 `_parent_` 匹配 |
